# Logic Line 前 30 分钟流程草案

本文档定义玩家从空白地图开局到第一次击败阶段性巢穴守军与小 Boss 的体验顺序。目标不是展示所有系统，而是让玩家按最小认知负担理解核心闭环：

`采集资源 -> 建立生产 -> 组装机器人 -> 观察失败 -> 修改逻辑 -> 集结突破 -> 解锁下一阶段`

---

## 一、对当前五个回答的设计判断

### 1. 从空地图建立基地与资源采集开始

**判断：可行，但需要降低开局噪音。**

这符合 RTS 和自动化游戏直觉，玩家不用先理解战斗逻辑就能进入熟悉的“放基地、采资源、造产线”状态。问题是你的游戏真正卖点是逻辑编程，如果前 10 分钟完全像普通自动化游戏，玩家可能会误判游戏类型。

**修正建议：**

- 开局地图只放 1 个铁矿、1 个铜矿、1 个敌巢方向标记。
- 只开放 4 个建筑：主基地、采矿机、基础加工厂、机器人锻造厂。
- 在 UI 上尽早展示“机器人逻辑槽”，但先锁住高级规则，让玩家知道后面会编辑 AI。
- 不要一开始开放复杂传送带网络，可以先用主基地范围内的自动物流半径完成基础供给。

### 2. 前 5 分钟生产基础物流机器人、加工厂和第一个作战单位

**判断：方向正确，但“物流机器人”和“作战机器人”的职责要清晰分离。**

如果玩家前 5 分钟同时学习资源、物流、加工、组装、战斗，会略重。物流机器人是好概念，但如果它也需要逻辑或路径规则，会把新手引导拉复杂。

**修正建议：**

- 前 5 分钟的物流机器人应当是“无脑默认服务单位”，不需要玩家配置。
- 玩家只需要理解：基础资源进主基地，主基地自动派物流机器人搬运到附近工厂。
- 第一个作战单位使用固定蓝图：轻型底盘 + 轻型步枪 + 基础逻辑板。
- 第一个作战单位只有默认脑干，不要求玩家编辑。

### 3. 第一次失败由最简单敌人造成，只伤建筑不摧毁建筑

**判断：非常适合教学，但要避免玩家觉得敌人没有威胁。**

低惩罚第一次失败是合理的。敌人只造成伤害、不摧毁建筑，可以让玩家观察问题而不是忙着救火。但如果表现太温和，玩家不会产生“我需要优化逻辑”的动机。

**修正建议：**

- 第一次袭击敌人只会把建筑打到 50% HP，并触发“自动紧急护盾”，不会摧毁。
- UI 明确提示：“基地受损，若下一波袭击前未解决敌巢，护盾不会再次保护。”
- 用战斗报告告诉玩家：`敌人抵达基地：3/5`，`我方机器人零散交战，未形成有效火力`。
- 第一次失败不是“输”，而是“系统警告”。

### 4. 第一次 Aha Moment：默认机器人只会找敌攻击，缺少集结动作

**判断：这是一个很好的第一课，比残血撤退更适合新手。**

“集结”比“复杂目标选择”更直观。玩家很容易看懂：机器人一台一台冲出去，被敌巢守军逐个吃掉；加上集结逻辑后，机器人先聚成一队，再一起推进，战斗结果立刻改变。

潜在问题是：当前规则系统还没有“集结点”“友军数量”“等待到 N 台再进攻”这类条件和动作。它会要求早期开发新增几个基础概念。

**建议把第一次 Aha Moment 设计为：**

默认脑干：

`If 发现敌人 -> Then 接近并开火`

玩家新增规则：

`If 周围友军数量 < 3 -> Then 前往集结点`

或更简化：

`If 未到达集结点 -> Then 前往集结点`

当 3 台以上机器人到达集结点后，默认脑干接管，队伍一起攻击敌巢。

这个 Aha Moment 的优点是：玩家只改一条规则，就能从“添油失败”变成“成队突破”。

### 5. 阶段结束条件是击败地图敌巢守军和 Boss

**判断：可行，是清晰的阶段目标。**

敌巢守军 + 小 Boss 能自然承载“考题”概念，也方便绑定科技解锁。需要注意的是，第一阶段 Boss 不应考验复杂战术，而应只考验玩家是否理解了生产持续性和集结逻辑。

**修正建议：**

- 第一阶段 Boss 不要有复杂护盾、反伤、状态切换。
- Boss 的机制只做一件事：单个机器人无法击败它，但 3 到 5 台同时攻击可以击败。
- 击败后掉落第一个关键科技材料，例如 `初级感应线圈`，解锁更多逻辑规则槽或 `友军数量` 条件。

## 二、前 30 分钟目标

### 玩家应该学会的内容

1. 资源必须被自动采集和加工。
2. 作战单位不是直接购买，而是由底盘、武器、逻辑板组成。
3. 默认脑干能打小怪，但不能解决所有战术问题。
4. 一条简单的 If-Then 规则可以改变战斗结果。
5. 击败敌巢是阶段性推进与科技解锁的核心目标。

### 玩家不应该在前 30 分钟学习的内容

- 多武器过热。
- 弹药补给。
- 复杂敌人标签。
- Boss 形态切换。
- 高级物流网络。
- 多队伍协同广播。

## 三、前 30 分钟时间线

### 0:00 - 2:00：落地与基地建立

**玩家行为：**

- 在空白地图中心放置主基地。
- 主基地自动扫描附近资源点。
- 玩家放置第一个采矿机到铁矿上。

**系统开放：**

- 主基地。
- 采矿机。
- 基础资源面板。

**教学目标：**

让玩家理解“基地是生产与防守核心”。

**风险：**

如果从零开始太慢，玩家会在看到核心玩法前流失。

**控制方式：**

主基地初始携带少量资源，足够建造第一台采矿机和基础加工厂。

### 2:00 - 5:00：基础生产启动

**玩家行为：**

- 建立铜矿采集。
- 建立基础加工厂。
- 加工铁板、铜线、简易齿轮。
- 主基地自动生产基础物流机器人，在基地范围内搬运材料。

**系统开放：**

- 基础加工厂。
- 简易配方：铁板、铜线、简易齿轮。
- 基础物流机器人。

**教学目标：**

让玩家看到“后勤自动化会变成战斗力”。

**风险：**

物流机器人过早出现可能让玩家误以为这是物流管理游戏，而不是逻辑战斗游戏。

**控制方式：**

物流机器人不提供自定义逻辑，不进入战场，只作为基地服务层存在。

### 5:00 - 8:00：第一台作战机器人

**玩家行为：**

- 建立机器人锻造厂。
- 生产第一台轻型步枪机器人。
- 机器人自动向最近敌人移动并开火。

**系统开放：**

- 机器人锻造厂。
- 固定蓝图：轻型底盘 + 轻型步枪 + 基础逻辑板。
- 默认脑干：寻敌、接近、开火。

**教学目标：**

让玩家第一次看到“自己生产的单位会自动战斗”。

**风险：**

如果第一台机器人就打赢所有敌人，逻辑系统会显得多余。

**控制方式：**

第一台机器人能打赢游荡小怪，但无法突破敌巢守军。

### 8:00 - 12:00：第一次低惩罚失败

**事件：**

玩家附近敌巢派出 3 到 5 个拾荒猎犬。它们攻击外围建筑或主基地，但不会摧毁建筑。

**玩家观察到的问题：**

- 机器人逐个冲出去。
- 敌人数量较多时，单台机器人容易被围殴。
- 若生产线速度不足，基地会被敌人摸到。

**系统反馈：**

- 建筑 HP 降低。
- 主基地触发一次性新手护盾。
- 战斗报告出现：
  - `袭击结果：基地受损`
  - `主要原因：我方单位零散接敌`
  - `建议：尝试让机器人先集结再进攻`

**教学目标：**

让玩家意识到“默认脑干能打，但不够聪明”。

### 12:00 - 16:00：解锁第一条自定义逻辑

**玩家行为：**

- 打开机器人蓝图界面。
- 看到默认脑干规则不可删除，但可以添加一条高级规则插队。
- 添加或选择预设规则：`先到集结点`。

**推荐 UI 表达：**

规则预设块：

`如果 [未到达集结点] -> [前往集结点]`

或进阶版本：

`如果 [周围友军数量 < 3] -> [前往集结点]`

**系统开放：**

- 逻辑面板。
- 集结点放置工具。
- 条件：是否到达集结点。
- 动作：前往集结点。

**教学目标：**

这是第一次 Aha Moment 的准备阶段：玩家只需要理解一条规则，不需要理解完整规则系统。

**开发备注：**

如果当前实现暂时没有集结点系统，可以用固定地图点位代替。第一版不需要玩家自由放置集结点，只要在敌巢前方放一个可见的 Rally Marker。

### 16:00 - 22:00：第二次进攻与 Aha Moment

**玩家行为：**

- 继续生产轻型机器人。
- 机器人不再逐个冲入敌巢，而是先移动到集结点。
- 当数量达到阈值或敌人进入视野后，机器人一同攻击。

**理想表现：**

- 玩家明显看到机器人形成小队。
- 敌巢守军被集火击败。
- 头顶或战斗日志显示规则触发：
  - `触发：前往集结点`
  - `集结完成：默认攻击逻辑接管`

**教学目标：**

让玩家获得“我改了一条逻辑，战斗结果改变了”的直接反馈。

**风险：**

如果集结规则太自动，玩家可能觉得自己没参与。

**控制方式：**

让默认配置无法击败守军，新增规则后明显改善结果。

### 22:00 - 30:00：阶段敌巢与小 Boss

**玩家行为：**

- 持续生产作战机器人。
- 维护基础资源供给。
- 观察机器人按集结逻辑推进至敌巢。
- 击败敌巢守军和小 Boss。

**Boss 设计：巢穴守卫者**

- 高 HP。
- 低移动速度。
- 没有复杂技能。
- 单台机器人无法击败。
- 3 到 5 台机器人同时攻击可以稳定击败。

**胜利奖励：**

- 掉落 `初级感应线圈`。
- 解锁基础逻辑控制板的完整功能。
- 解锁下一条条件或动作，例如：
  - `友军数量 < X`
  - `HP < X%`
  - `撤退`

**阶段结束条件：**

- 敌巢守军全灭。
- 小 Boss 死亡。
- 玩家获得第一个科技材料。

## 四、第一阶段最小系统需求

为了支撑上述流程，第一阶段至少需要以下系统。

### 已有或接近已有

- 机器人默认脑干。
- 距离判断。
- 开火、接近、停止。
- 机器人生产或生成。
- JSON 逻辑配置。
- 敌我队伍。

### 需要补充

- 基础资源与建筑放置。
- 主基地。
- 基础物流机器人，或临时自动搬运机制。
- 机器人锻造厂。
- 敌巢与守军。
- 新手护盾或低惩罚建筑受损机制。
- 集结点。
- 动作：前往集结点。
- 条件：未到达集结点，或周围友军数量小于 X。
- 战斗报告或规则触发反馈。

---

## 五、第一阶段的核心考题

第一阶段不考复杂战术，只考一个问题：

**玩家能否理解“零散出兵会失败，集结后进攻会成功”？**

这道考题足够承载游戏的核心承诺：

- 生产线提供持续兵力。
- 默认 AI 提供基础战斗力。
- 自定义逻辑提供战术突破。
- 敌巢提供阶段目标。
- 胜利奖励解锁下一层自动化与逻辑能力。

---

## 六、待定问题

1. 集结点是玩家手动放置，还是第一阶段固定地图点位？
2. 机器人到达集结点后，是什么条件让它们开始进攻？
3. “周围友军数量”统计半径是多少？
4. 物流机器人是否需要实体化，还是先作为主基地抽象搬运能力？
5. 第一阶段 Boss 掉落物用于解锁逻辑插槽，还是解锁新建筑？
6. 第一次袭击的敌人是否从敌巢真实出发，还是作为脚本事件生成？
