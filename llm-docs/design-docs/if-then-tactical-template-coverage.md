# If-Then 战术模板覆盖设计

本文档整理此前设计文档中出现过的 If-Then 规则期望场景，并把它们映射到更易使用的“战术模板”表达。

核心判断：

`战术模板不是枚举每一种战术，而是把常见战术意图压缩成少数可参数化的模板族。`

底层 If-Then 仍然存在，但玩家优先操作模板。模板可以在内部编译成多条规则、运行时阶段、战术标记或复合动作。玩家不应为了表达“先集结，再进攻”而手工维护 4 条规则和 2 个状态变量。

---

## 一、需要开发的战术模板总表

### 1. 默认脑干模板：`DefaultBrain`

用途：所有机器人保底行为。

玩家表达：

`没有特殊战术时，按默认行为行动`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 默认目标 | 最近敌人 / 敌巢 / 当前任务 |
| 默认移动 | 接近并保持射程 |
| 默认攻击 | 使用可用主武器 |
| 无目标行为 | 待命 / 巡向敌巢 / 返回岗位 |

底层含义：

`寻找目标 -> 接近目标 -> 进入射程后开火`

设计作用：让自定义规则只负责“特殊情况”和“战术意图”，而不是从零定义完整 AI。

### 2. 阶段流程模板：`SequenceTactic`

用途：表达“先做 A，再做 B，再交给默认脑干”的顺序行为。

玩家表达：

`先完成某个阶段，再进入下个阶段`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 阶段列表 | 前往集结点、等待队友、进攻 |
| 阶段完成条件 | 到达位置 / 人数达标 / 货仓已满 |
| 阶段失败处理 | 超时后进攻 / 返回基地 / 待命 |
| 完成后行为 | 默认脑干接管 / 执行指定模板 |

底层含义：

内部维护阶段状态，不要求玩家显式创建 `rallied`、`squad_ready` 等变量。

### 3. 集结推进模板：`RallyThenAttack`

用途：MVP 第一课，解决机器人零散送死。

玩家表达：

`先到集结点，等队友够了再一起进攻`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 集结点来源 | 锻造厂集结点 / 前哨集结点 / 手动标记 |
| 到达半径 | 20 |
| 等待友军数量 | 3 或 4 |
| 统计范围 | 集结点附近 / 自身附近 |
| 满足后行为 | 默认脑干接管 / 攻击敌巢 |
| 超时策略 | 继续等待 / 人数不足也进攻 / 返回锻造厂 |

模板内部流程：

1. 有集结点且未到达时，前往集结点。
2. 到达后进入等待状态。
3. 友军数量达到阈值后，放行默认脑干或指定进攻目标。

### 4. 目标选择模板：`TargetSelector`

用途：把“攻击谁”从底层规则中抽出来。

玩家表达：

`优先攻击符合这些特征的目标`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 目标池 | 敌人 / 建筑 / 掉落物 / 受损友方建筑 |
| 排序方式 | 最近 / 最远 / 血量最低 / 威胁最高 / 价值最高 |
| 标签过滤 | healer、sniper、heavy_armor、jammer、boss |
| 状态过滤 | 正在蓄力 / 开启护盾 / 已锁定 / 被冰冻 |
| 找不到目标 | 回落最近敌人 / 默认脑干 / 待命 |

底层含义：

模板输出一个 `current_target` 或行动目标，后续攻击、移动、广播、回收模板都可以复用。

### 5. 条件反应模板：`ReactiveOverride`

用途：处理高优先级紧急反应，如低血、过热、蓄力、基地受损。

玩家表达：

`如果出现危险情况，临时覆盖当前行为`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 触发条件 | HP 低于 30%、热量高于 80%、敌人蓄力、建筑受损 |
| 反应动作 | 撤退、停火、找掩体、回防、开护盾 |
| 恢复条件 | HP 恢复、热量降低、危险消失、抵达补给点 |
| 恢复后行为 | 返回原战术 / 默认脑干接管 / 待命 |

底层含义：

高优先级模板，可临时打断 `RallyThenAttack`、`DefaultBrain` 或其它流程模板。

### 6. 生存撤退模板：`RetreatAndRecover`

用途：低血、低护盾、寿命不足、弹药/能量不足时保命或回补。

玩家表达：

`状态不好时撤退到安全地点，恢复后再行动`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 触发条件 | HP < 30%、护盾 = 0、弹药不足、能量不足 |
| 目的地 | 修理坪 / 补给点 / 基地物流入口 / 安全区 |
| 撤退动作 | 停止开火、背离敌人、开护盾、释放诱饵 |
| 完成条件 | HP > 70%、补给完成、脱离危险 |
| 完成后行为 | 返回战场 / 默认脑干 / 继续物流任务 |

### 7. 站位与拉扯模板：`RangeStance`

用途：表达“保持射程”“风筝”“近战冲锋”“贴近后攻击”。

玩家表达：

`面对某类目标时保持指定距离或移动姿态`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 姿态 | 保持最大射程 / 后撤拉扯 / 冲锋近战 / 原地架枪 |
| 目标 | 当前目标 / 标签目标 / Boss |
| 期望距离 | 武器射程 85% / 近战范围 / 脱离雷达 |
| 触发状态 | 目标有护盾 / 目标护盾消失 / 敌人是自爆虫 |

底层含义：

移动意图与开火意图分离，避免机器人进入射程后继续贴脸。

### 8. 武器控制模板：`WeaponControl`

用途：处理过热、弹药、冷却、停火、主动武器触发。

玩家表达：

`在合适条件下使用或停止某个武器`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 武器槽 | 主武器 / 副武器 / 主动技能 |
| 开火条件 | 目标在射程内、目标被冰冻、收到锁定信号 |
| 停火条件 | 热量 > 80%、目标反伤护盾开启、弹药不足 |
| 冷却期间行为 | 后撤 / 机枪掩护 / 默认脑干 |

### 9. 连招模板：`ComboTactic`

用途：表达多武器或多单位协同的阶段性攻击。

玩家表达：

`先控制，再爆发；先锁定，再导弹；先牵引，再自毁`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 步骤 1 | 冰冻目标 / 标记目标 / EMP 瘫痪 |
| 步骤 2 | 发射导弹 / 近战链锯 / 重火力集火 |
| 步骤 3 | 停止充能 / 后撤 / 等冷却 |
| 失败处理 | 回到步骤 1 / 默认攻击 / 撤退 |

底层含义：

这是 `SequenceTactic` 的战斗版，可以跨武器，也可以跨蓝图通过广播与标记协同。

### 10. 防守回防模板：`DefendAsset`

用途：资源点、前哨、主基地、生产线受损时自动防守。

玩家表达：

`当重要资产受威胁时，放弃当前任务并回防`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 保护对象 | 主基地 / 前线补给点 / 采矿点 / 锻造厂 |
| 触发条件 | 对象受攻击 / HP < 50% / 雷达发现敌人 |
| 集结位置 | 前哨集结点 / 受损建筑附近 |
| 参战门槛 | 附近友军 >= 3 / 立即回防 |
| 完成条件 | 附近敌人清空 / 建筑修复 |

### 11. 回收返航模板：`SalvageAndReturn`

用途：战地拾荒、残骸回收、Boss 掉落物回收。

玩家表达：

`发现值得捡的东西就回收，满载或危险时返回`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 回收目标 | 稀有掉落物 / 我方残骸 / Boss 掉落 / 弹药箱 |
| 优先级 | 稀有度 / 距离 / 安全程度 / 任务指定 |
| 进入条件 | 周围敌人少 / 护盾充足 / 有装甲底盘 |
| 返回条件 | 背包满 / HP 低 / 发现敌人过多 / 拿到关键物 |
| 返回地点 | 基地物流入口 / 前线补给点 / 科技建筑 |

### 12. 物流补给模板：`SupplyRun`

用途：弹药、维修材料、能量包、远程投送。

玩家表达：

`前线缺什么，就把指定物资送过去`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 需求来源 | 前线补给点 / 建筑 / 导弹车 / 维修点 |
| 物资类型 | 弹药箱 / 维修材料 / 电池 / 集结信标 |
| 运输方式 | 普通搬运 / 远程投送 / 投送无人机 |
| 危险策略 | 敌人过多则撤离 / 释放诱饵 / 等待护航 |

### 13. 广播协同模板：`SignalCoordination`

用途：后期多蓝图协同。

玩家表达：

`我发现重要目标时发信号；队友收到信号后行动`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 发送条件 | 发现干扰塔 / 标记 Boss / 发现高价值残骸 |
| 信号内容 | 目标 ID / 位置 / 目标标签 / 求援 |
| 接收对象 | 导弹车 / 回收车 / 防守队 / 全队 |
| 接收动作 | 发射导弹 / 前往位置 / 回防 / 回收 |
| 信号有效期 | 5 秒 / 直到目标死亡 / 直到取消 |

### 14. 目标标记模板：`MarkAndExploit`

用途：目标锁定器、巡航导弹、高价值目标集火。

玩家表达：

`先给目标打标，再让特定单位利用标记`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 标记目标 | 威胁最高 / 血量最高 / 干扰塔 / Boss |
| 标记类型 | 已锁定 / 高价值 / 需回收 / 需集火 |
| 利用方式 | 导弹发射 / 集火 / 共享视野 / 回收 |
| 标记过期 | 目标死亡 / 超时 / 失去视野 |

### 15. 危险规避模板：`HazardAvoidance`

用途：激光蓄力、地雷、酸液、敌人过多、战场危险区。

玩家表达：

`发现危险就躲避或绕行`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 危险类型 | 蓄力激光 / 自爆虫群 / 地雷 / 敌人数量 > 2 |
| 规避动作 | 找掩体 / 后撤 / 绕行 / 释放诱饵 |
| 恢复条件 | 危险消失 / 离开范围 / 技能结束 |

### 16. 终局牺牲模板：`LastResort`

用途：寿命将尽、自毁、反物质炸弹、战略换命。

玩家表达：

`反正快报废了，就用最后动作换取最大收益`

典型配置：

| 参数 | 示例 |
| --- | --- |
| 触发条件 | 寿命 < 5%、HP < 10%、被包围 |
| 目标条件 | 周围敌人 > 5、靠近敌方核心、Boss 在范围内 |
| 动作 | 引爆奇点炸弹 / 冲向核心 / 广播最终目标 |
| 安全限制 | 禁止友军过近 / 只对 Boss 区域启用 |

## 二、模板与底层规则的关系

模板不要求一对一对应一条底层 If-Then。建议有三种编译方式：

| 编译方式 | 适用场景 | 说明 |
| --- | --- | --- |
| 单条规则 | 简单反应 | 例如 `HP < 30% -> 撤退` |
| 多条规则 | 当前已有规则系统可承载的流程 | 例如集结、回收、低血返回 |
| 复合动作/运行时状态机 | 玩家不应手写的顺序行为 | 例如 `RallyThenAttack`、`ComboTactic` |

玩家 UI 默认显示模板，提供“展开高级规则”入口。展开后可以看到模板生成的底层规则，但 MVP 不要求玩家编辑展开内容。

## 三、已有设计场景覆盖表

### 1. 默认机器人攻击

来源文档：`game-design-document.md`、`first-playable-mvp.md`、`robot-blueprint-flow.md`

原始意图：

`发现敌人 -> 接近并开火`

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `默认脑干：发现敌人后接近并开火` |
| 模板族 | `DefaultBrain` |
| 关键参数 | 默认目标 = 最近敌人；移动 = 保持武器射程；攻击 = 主武器 |
| 底层规则 | 没有自定义规则命中时自动执行 |

设计备注：

默认脑干不占规则槽，不可删除。它让玩家只需要添加能改变战局的特殊模板。

### 2. MVP 先集结再进攻

来源文档：`first-30-minutes.md`、`first-playable-mvp.md`、`logic-rules-2.md`、`tech-tree.md`

原始意图：

机器人一台一台冲向敌巢会失败，集结 3-5 台后一起进攻可以突破。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `集结后进攻：到锻造厂集结点，等待 4 台友军，然后进攻` |
| 模板族 | `RallyThenAttack` |
| 关键参数 | 集结点 = 锻造厂集结点；到达半径 = 20；等待友军 = 4；完成后 = 默认脑干 |
| 底层展开 | 前往集结点；到达后等待；人数达标后放行默认脑干 |

设计备注：

这是最应该模板化的场景。玩家不应看见 `rallied`、`squad_ready` 这两个底层状态变量。

### 3. 简化版只前往集结点

来源文档：`first-playable-mvp.md`、`first-30-minutes.md`

原始意图：

第一版可以用更稳定的规则：未到达锻造厂集结点时前往集结点，然后默认脑干接管。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `先到集结点：出生后先前往集结点` |
| 模板族 | `SequenceTactic` |
| 关键参数 | 阶段 1 = 前往集结点；完成后 = 默认脑干 |
| 底层展开 | 如果有集结点且未到达，移动到集结点；到达后结束模板 |

设计备注：

适合作为 MVP 的最小可用实现，比完整“等人数”少一个门槛。

### 4. 目标优先级：刺客切后排

来源文档：`logic-rules-1.md`、`game-design-document.md`

原始意图：

敌方有前排重装坦克和后排奶妈/狙击手，默认攻击最近敌人会失败。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `优先目标：攻击带 healer 或 sniper 标签的敌人` |
| 模板族 | `TargetSelector` + `RangeStance` / `DefaultBrain` |
| 关键参数 | 标签过滤 = healer/sniper；找不到目标 = 默认最近敌人；动作 = 接近并攻击 |
| 底层展开 | 选择符合标签的最高优先级目标，并覆盖默认目标 |

可选扩展：

`若目标距离 < 近战范围，则使用链锯`

对应模板：

`WeaponControl`，开火条件 = 当前目标在近战范围内。

### 5. 避开重甲，锁定高输出敌人

来源文档：`logic-rules-2.md`、`production.md`

原始意图：

基础机器人只打最近敌人，面对“前排重甲 + 后排输出”会吃瘪。弱点分析处理器解锁后，可以忽略重甲并锁定高输出目标。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `弱点优先：忽略重甲，攻击威胁最高敌人` |
| 模板族 | `TargetSelector` |
| 关键参数 | 排除标签 = heavy_armor；排序 = 威胁最高 / 输出最高；找不到目标 = 默认脑干 |
| 科技依赖 | 弱点分析处理器 / 中高级逻辑板 |

### 6. Boss 护盾形态切换

来源文档：`logic-rules-1.md`

原始意图：

Boss 形态 A 开启反射远程护盾但怕近战；形态 B 护盾消失后近战危险但怕远程。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `形态应对：目标开盾时停火并近战，护盾消失后拉开远程` |
| 模板族 | `ReactiveOverride` + `RangeStance` + `WeaponControl` |
| 关键参数 | 条件 1 = 目标有 shield_reflect；动作 = 停止远程、接近、使用近战 |
| 关键参数 | 条件 2 = 目标无 shield_reflect；动作 = 保持射程、启用远程 |

设计备注：

UI 不应要求玩家写两套互斥规则。可显示为一个“按目标状态切换姿态”的模板。

### 7. 自适应巨兽伤害类型切换

来源文档：`enemy-docs.md`

原始意图：

巨兽受到某种伤害过多后生成对应免疫护甲。玩家必须让不同武器部队协同切换。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `自适应反制：目标开启能量护甲时停用激光，动能部队集火` |
| 模板族 | `ReactiveOverride` + `WeaponControl` + `SignalCoordination` |
| 关键参数 | 目标状态 = energy_armor；激光单位动作 = 停火/后撤；动能单位动作 = 接收信号并攻击 |
| 后期扩展 | 通过广播把 Boss 当前抗性同步给多蓝图部队 |

### 8. 敌人蓄力激光时找掩体

来源文档：`logic-rules-1.md`

原始意图：

敌人的激光主炮有蓄力预警，机器人需要躲避。

模板写法：

| 层级 | 写法 |
| --- | --- |
| 玩家模板 | `危险规避：敌人蓄力时寻找掩体` |
| 模板族 | `HazardAvoidance` |
| 关键参数 | 危险类型 = enemy_casting_laser；规避动作 = 前往最近掩体；恢复 = 蓄力结束 |

## 四、推荐 MVP 收敛方案

MVP 不要一次开发全部模板。建议首版只开发：

1. `DefaultBrain`
2. `RallyThenAttack`
3. `TargetSelector` 的最小版本
4. `ReactiveOverride` 的最小版本
5. `RetreatAndRecover` 的低血撤退版本，暂可不进首个试玩

MVP 最关键的是 `RallyThenAttack`。它应该直接替代当前需要 4-5 条底层规则的集结流程。

### MVP UI 建议

蓝图规则区默认显示：

`战术模板`

可选项：

- `默认进攻`
- `先到集结点`
- `集结后进攻`

选择 `集结后进攻` 后显示参数：

| 参数 | 默认值 |
| --- | --- |
| 集结点 | 当前锻造厂集结点 |
| 等待友军 | 4 |
| 到达半径 | 20 |
| 完成后 | 默认进攻 |

高级按钮：

`展开底层规则`

展开后只读显示内部状态和规则，第一版不允许新手直接编辑。

## 五、长期 UI 结构建议

模板 UI 可以采用“意图句子 + 参数槽”的形式：

`当 [自身状态不好] 时 [撤退到补给点]，直到 [恢复完成]`

`优先选择 [带 healer 标签的敌人]，然后 [接近并攻击]`

`发现 [干扰塔] 时 [广播锁定目标]`

`收到 [已锁定目标] 时 [发射巡航导弹]`

这样玩家看到的是战术语言，而不是：

`IF A AND B AND C THEN SET flag_x = true`

底层规则仍然保留给：

- 调试。
- 高级玩家。
- 模板展开。
- 后期终极 AI 矩阵。

---

## 六、设计结论

此前设计文档中的 If-Then 场景基本可以被 16 个模板族覆盖，不需要为每个敌人或每个关卡枚举一个独立战术模板。

最重要的区分是：

- `模板族`：少数、稳定、可开发。
- `模板参数`：大量、由模块、敌人、地图和科技提供。
- `底层规则`：作为实现细节和高级编辑入口，不作为新手默认表达。

因此，战术模板方案可以覆盖既有设计目标，但实现时必须避免把模板做成“完整 AI 预设列表”。真正要做的是一套面向玩家意图的战术语法。
