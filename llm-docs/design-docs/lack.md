我读完四份文档后，感觉现在的设计已经有了很强的“系统想象力”：生产链、敌人生态、武器底盘、If-Then 逻辑都很丰满。但从**整体游戏流程**看，还缺少一些把这些系统串成“玩家每天点开就知道要干嘛”的流程层设计。

**最大的缺口：从系统集合到关卡节奏**
现在文档描述了很多“系统能做什么”，但还缺少“玩家按什么顺序体验它们”。比如 [game-design-document.md](game-design-document.md) 里有大循环：采矿、生产、配置、攻克巢穴、解锁科技。但还没有明确：

- 第一局玩家从哪里开始？
- 前 5 分钟只需要做什么？
- 第一次失败应该由什么敌人造成？
- 玩家第一次修改逻辑规则的“顿悟点”是什么？
- 每个阶段的结束条件是什么？

建议补一份 **campaign-flow.md / first-30-minutes.md**，专门定义新手到中期的节奏。

**缺少明确的胜负条件**
目前有敌巢、Boss、主脑核心、基地骚扰，但还没有清晰的失败与胜利规则。

需要补：

- 玩家基地核心被摧毁是否失败？
- 敌人是否会无限进化，还是每个巢穴独立？
- 地图目标是摧毁所有巢穴、击败主脑，还是达成自动化生产指标？
- 单场战斗失败后损失什么？机器人、资源、产线、时间、威胁值？
- 机器人寿命到期回收，是否返还材料？返还比例是多少？

没有这些，生产链和敌人压力会很难平衡。

**缺少“部署”这一层的具体设计**
文档里生产和战斗都很强，但中间的“机器人如何从工厂进入战场”还不够明确。

要补：

- 机器人锻造厂产出的机器人是自动走向前线，还是进入缓冲队列？
- 玩家是否能指定出生点、战线方向、集结区？
- 多条产线如何对应多支队伍？
- 不同规则 JSON/蓝图如何绑定到某条生产线？
- 机器人死亡或寿命结束后，产线是否自动补位？

这个环节很关键，因为它决定游戏到底像《异星工厂》、自走棋，还是自动 RTS。

**缺少机器人蓝图生命周期**
[production.md](production.md) 和 [weapon-chasis-docs.md](weapon-chasis-docs.md) 里已经有很多底盘、武器、逻辑板，但还缺少“蓝图对象”的规则。

建议定义：

- 一个机器人蓝图包含：底盘、武器槽、逻辑板、规则文件、生产配方、部署策略。
- 蓝图能否保存、复制、升级？
- 蓝图修改后，已经在战场上的机器人是否改变？
- 产线生产的是“当前蓝图快照”还是实时引用？
- 逻辑插槽数量如何限制规则数量？

这会直接影响 UI、存档和生产系统。

**缺少资源经济的数值节奏**
文档有大量配方，但还没有经济节奏指标。

需要补：

- 每种矿每分钟基础产量是多少？
- 一个基础机器人生产周期是多少？
- 前期玩家合理状态下每分钟能生产几台机器人？
- 敌人威胁值增长速度是多少？
- 敌人袭击频率如何随工厂规模变化？
- 掉落物是必掉、概率掉，还是巢穴奖励？

否则很容易出现两个极端：玩家爆兵碾压，或者生产线永远跟不上。

**缺少敌人进攻与巢穴目标流程**
[enemy-docs.md](enemy-docs.md) 的敌人生态方向很好，尤其是威胁预算、集结点、小队推进。但还需要流程化：

- 敌巢如何被发现？
- 敌巢会主动扩张吗？
- 玩家攻击巢穴时，巢穴如何切换到防守模式？
- 敌人袭击目标选择：矿机、电网、传送带、核心，优先级是什么？
- 小队袭击成功后会撤退、拆家，还是战斗到死？
- 玩家摧毁巢穴后，威胁值如何变化？

这部分会决定地图是不是“活的”。

**缺少科技树解锁图**
现在各文档里散落着解锁内容：感应线圈、振荡器、能量传导液、量子处理器、主脑核心。但还没有一张明确的科技树。

建议补：

- 每个科技节点需要什么材料/掉落物？
- 解锁什么建筑、武器、底盘、逻辑条件、动作？
- 哪个敌人是哪个科技节点的门槛？
- 玩家是否可以跳科技？
- 科技节点是线性、分支，还是地图驱动？

这会把“敌人是考题”真正落到进程上。

**缺少逻辑系统的可用性保护**
[logic-rules-1.md](logic-rules-1.md) 已经提到默认脑干和下拉菜单，但整体流程还需要防呆规则：

- 没写规则时使用默认脑干。
- 写了无效规则时如何提示？
- 条件参数为空时如何处理？
- 规则数量超过逻辑板插槽时怎么办？
- 多个动作是否允许同一条规则执行？
- 被 EMP 屏蔽前两条规则时，默认脑干是否还能工作？

这些决定玩家会不会觉得是在“调 bug”，而不是玩策略。

**缺少战斗反馈与复盘**
目前文档提到触发逻辑时头顶气泡，这是好方向，但整体游戏还需要复盘层：

- 本轮生产了多少机器人？
- 死亡原因统计：被击杀、寿命耗尽、过热、弹药耗尽。
- 哪条规则触发最多？
- 哪条规则从未触发？
- 哪种敌人造成最多损失？
- 资源消耗与击杀收益是否划算？

这个系统非常适合你的游戏，因为玩家的核心乐趣就是“调参变强”。

**建议优先补的 5 份文档**
1. `first-30-minutes.md`：新手流程和第一次 Aha Moment。
2. `victory-loss-and-map-flow.md`：胜负、巢穴、地图推进、失败惩罚。
3. `robot-blueprint-flow.md`：蓝图、规则文件、产线、部署绑定关系。
4. `tech-tree.md`：敌人掉落到科技解锁的完整链路。
5. `combat-report-and-debug-ui.md`：战斗复盘、规则触发反馈、损失统计。

我会把当前状态概括成一句话：**你已经有了很棒的系统零件，但还缺少“玩家旅程胶水”。** 下一步最应该做的不是继续加武器或敌人，而是定义玩家从开局到第一次击败巢穴之间，每一分钟在做什么、为什么失败、如何靠修改逻辑赢回来。
