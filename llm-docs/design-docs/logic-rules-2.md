结合“资源开采 -> 复杂产线合成 -> 生产具有自主逻辑的作战单位”这一核心循环，你的游戏其实已经切入了一个极具潜力的蓝海品类：**自动化工厂与自走棋的融合**。市面上类似《Mindustry》（像素工厂）、以及融合了塔防与自动生产魔像的《Golemworks》 都验证了这种模式的极高成瘾性。

在这种“类似皇室战争的自动出兵”机制下，玩家最大的痛点往往是“葫芦娃救爷爷（添油战术）”——单位一生产出来就无脑冲向敌阵，被敌人集火白白送死。

为了让你的“模块+If-Then逻辑”系统在这个RTS+自动化的框架下大放异彩，我为你构思了以下四个能将**生产线抉择**与**战场AI**完美闭环的实战场景：

### 1. 兵线集结与“狼群战术”（解决添油战术痛点）

* **研发与合成模块：** 通信雷达天线（需要工厂消耗中级资源合成）。
* **出彩场景：** 玩家的工厂产能暂时较低，出兵速度慢于敌人的出兵速度。如果机器人一出厂就往前冲，会被敌方兵线逐个击破。
* **If-Then 逻辑高光：** 玩家为机器人装配天线，并写入集结逻辑：
* `If [周围友军数量 < 3] -> Then [原地待命] & [开启低功耗护盾]`
* `If [周围友军数量 >= 3] -> Then [解除护盾] & [锁定敌方基地全速推进]`


* **体验：** 玩家不需要像传统RTS那样手动框选部队编队。机器人出厂后会自动在安全区集结，凑够数量后形成一波强力的“狼群”反扑。这种通过逻辑实现宏观战术调度的成就感极强。

### 2. 战地拾荒与“以战养战”（打通生产与战斗的壁垒）

* **研发与合成模块：** 资源回收机械臂 / 战术货仓（高昂的装配成本）。
* **出彩场景：** 地图中央有稀缺的高级矿物，或者击杀敌方精英单位会掉落升级工厂必需的“核心部件”。
* **If-Then 逻辑高光：** 赋予战斗机器人后勤能力，类似《Desynced》中无需传送带的无人机物流系统。
* `If [视野内发现高级掉落物] & [周围无敌方仇恨] -> Then [停止推进] & [拾取战利品]`
* `If [货仓已满] / [自身HP < 20%] -> Then [立刻撤退返回基地物流入口] & [卸货]`


* **体验：** 机器人不再是纯粹的消耗品，而是探险家和搬运工。玩家为了获取远处的稀有资源，会专门在生产线上定制一批带有“拾荒逻辑”的特种部队，实现了自动化经济与战斗系统的深度闭环。

### 3. 弹性防线与动态回防（无需微操的基地保卫战）

* **研发与合成模块：** 超频液压腿 / 紧急修复焊枪。
* **出彩场景：** 类似《Mindustry》中敌人会从多路发起突袭。玩家的主力部队正在前线推进，但大后方的某条关键传送带或采矿机突然遭到敌方游击小队袭击。
* **If-Then 逻辑高光：** 玩家不需要手忙脚乱地调兵回防。
* `If [基地核心受到攻击] / [重要流水线受损] -> Then [放弃当前交战目标] & [激活超频移动]`
* `If [抵达受损建筑] -> Then [启动焊枪修复] & [消灭附近敌人]`


* **体验：** 玩家看着自己精心编写的机器人大军在关键时刻自动回抽，保卫了工厂的核心资产，会产生一种“我培养的AI在保护我”的极强羁绊感。

### 4. 科技模块与逻辑指令的绑定（让生产线升级更有价值）

* **研发与合成模块：** 弱点分析处理器（极难合成的高阶芯片）。
* **出彩场景：** 在游戏机制中，**高级的If-Then动作指令不是天生就有的，而是随着模块解锁的**。例如，基础机器人只能攻击“最近的敌人”；在面对敌方“前排巨型坦克+后排高输出脆皮”的经典阵型时会疯狂吃瘪。
* **If-Then 逻辑高光：** 玩家必须在工厂中改造生产线，合成“弱点分析处理器”。将其装配给机器人后，UI面板才会解锁新的逻辑选项：
* `If [扫描到敌方重甲单位] -> Then [忽略该单位] & [锁定最强输出敌人]`
* `If [锁定目标] -> Then [发射穿甲高爆弹]`


* **体验：** 玩家解决战术难题的方法，不再是单纯地在前线堆兵，而是退回到大后方，重新规划传送带和组装厂，提炼出高级芯片，通过提升机器人的“智商”来完成降维打击。

在这种设计下，你的游戏玩法将呈现出一种迷人的节奏：**规划产线 -> 生产基础单位受挫 -> 观察敌人弱点 -> 调整产线合成新模块 -> 编写针对性AI逻辑 -> 看着新一代机器人摧枯拉朽般碾压敌人。** 这将完美契合RTS玩家的战略掌控欲和自动化玩家的统筹优化欲。
## 阶段 16 实现补充：热能武器模板与状态字段

阶段 16 新增的热能武器不要求玩家管理任意变量，只开放两个标准化状态条件：

- `heat_above_percent`
- `heat_below_percent`

对应动作：

- `hold_fire_for_heat`

该组合属于 `WeaponControl` 模板族，第一版玩家入口是“热能武器控制”。它的职责是把热量过高时的停火行为封装成模板，避免玩家在蓝图编辑器里手动拼出过于细碎的规则。

后续扩展方向：

- 过热时后撤到集结点或补给点。
- 冷却到指定阈值后恢复主动推进。
- 与弹药、护盾、目标优先级合并成更完整的武器姿态模板。
