# 科技树与阶段解锁设计

## 设计目标

科技树的作用不是单纯延长游玩时间，而是把“敌人是考题”落实到游戏流程中：

- 玩家先用当前阶段的生产、机器人蓝图、逻辑规则和物流能力解决一个明确问题。
- 阶段性巢穴与 Boss 掉落关键研究材料。
- 研究建筑消耗中间产物与敌人掉落物，产出全局科技解锁结果。
- 下一阶段开放新的建筑、底盘、武器、逻辑条件、动作和数值升级。

整体结构采用“阶段锁 + 阶段内分支”：

- 不同阶段之间不能跳科技。
- 每个阶段由 Boss 掉落物作为硬门槛。
- 同一阶段内的科技节点可以在满足依赖关系后自由选择。
- 每个阶段的内容规模应保持可读，建议不超过 10 个核心科技节点、20 种左右新增物品或模块。

## 初始可用内容

游戏最小可行范围不依赖科技树，玩家开局即可使用：

| 类型 | 初始内容 | 作用 |
| --- | --- | --- |
| 核心建筑 | 主基地 | 存储基础资源，被动生产建设质料，提供基础建造入口 |
| 资源建筑 | 基础采矿机 | 采集铁矿、铜矿等基础资源 |
| 生产建筑 | 基础加工厂、部件装配机、机器人装配台 | 生产早期部件与基础机器人 |
| 资源 | 铁矿、铜矿、建设质料 | 支撑第一个生产闭环 |
| 物流机器人 | 默认微型物流底盘 | 无需配置，负责短距离搬运 |
| 战斗机器人 | 轻型战斗底盘 + 基础动能武器 | 生产第一个作战单位 |
| 逻辑 | 默认保底逻辑 | 寻找敌人并攻击，不包含复杂战术 |

这些内容用于保证玩家能完成第一条完整体验链：

建设主基地 -> 采集资源 -> 加工部件 -> 生产物流机器人 -> 生产战斗机器人 -> 进攻第一处弱巢穴。

## 研究建筑

建议新增建筑：研发终端。

研发终端的职责：

- 消耗中间产物、建设质料和敌人掉落物。
- 产出全局科技解锁，不直接产出普通物品。
- 允许玩家在同一阶段内选择研究顺序。
- 展示当前阶段 Boss 门槛和下一阶段入口。

研究结果建议作为存档中的全局解锁状态，而不是可搬运物品。敌人掉落物和中间产物仍然是实体资源，可以由物流机器人搬运到研发终端。

## 阶段总览

| 阶段 | 阶段主题 | 硬门槛掉落物 | 玩家需要学会的能力 |
| --- | --- | --- | --- |
| 阶段 0：开局工业 | 从零建立生产闭环 | 无 | 采矿、加工、建造、短距离物流、生产第一批战斗机器人 |
| 阶段 1：基础战术 | 从自动攻击到集结进攻 | 初级感应线圈 | 使用集结、数量判断和第一批自定义逻辑 |
| 阶段 2：复合生产 | 对抗装甲与空中骚扰 | 高频振荡器、能量传导液 | 生产弹药/能源链，使用更重底盘和基础物流扩展 |
| 阶段 3：战场回收 | 对抗干扰与高价值残骸 | 量子处理器 | 回收残骸、使用广播逻辑、远距离补给 |
| 阶段 4：终局攻坚 | 对抗主脑级敌人 | 主脑核心 | 组织多蓝图部队、远程投送、终极武器与终局科技 |

## 阶段 1：基础战术

开启条件：击败第一处弱巢穴守军或小型巢穴 Boss，获得初级感应线圈。

阶段目标：让玩家第一次意识到默认机器人“只会找敌人攻击”并不够，必须学会集结与简单规则配置。

| 科技节点 | 需求材料 | 解锁内容 | 设计作用 |
| --- | --- | --- | --- |
| 基础逻辑板 | 初级感应线圈、基础电路、建设质料 | 1 个逻辑插槽、规则编辑入口 | 开启自定义规则系统 |
| 集结信标 | 初级感应线圈、铜线圈、铁板 | 集结点建筑/动作 | 解决零散出兵被逐个消耗的问题 |
| 友军数量判断 | 初级感应线圈、基础电路 | 条件：附近友军数量 | 让玩家写出“人数足够再进攻”的第一条规则 |
| 基础战斗报告 | 初级感应线圈、基础传感器 | 显示机器人死亡、伤害、目标分布 | 帮助玩家判断规则是否有效 |
| 小型仓储 | 铁板、建设质料 | 小型资源仓库 | 支持早期生产线缓冲 |
| 轻型物流容量升级 | 初级感应线圈、基础背包模块 | 默认物流机器人容量 +1 或解锁小背包模块 | 平滑引入物流机器人升级 |

阶段 Boss 门槛建议：

- 敌人类型：低血量近战群、少量远程单位。
- 失败原因：机器人零散冲入巢穴，被数量优势消耗。
- 正确解法：先集结，达到数量阈值后一起进攻。

## 阶段 2：复合生产

开启条件：击败装甲型巢穴 Boss 和能量型巢穴 Boss，获得高频振荡器与能量传导液。

阶段目标：引入更复杂的生产链、装甲敌人、轻量空中骚扰和战场补给压力。

| 科技节点 | 需求材料 | 解锁内容 | 设计作用 |
| --- | --- | --- | --- |
| 强化冶炼 | 高频振荡器、钢坯、建设质料 | 强化冶炼炉、强化钢板 | 支撑中期底盘和建筑 |
| 高容量电池 | 能量传导液、铜线圈、基础电路 | 电池中间产物、能量武器供能 | 打开能量链 |
| 标准货运底盘 | 高频振荡器、强化钢板、背包模块 | 标准物流机器人底盘、更多背包栏位 | 解决中距离物流压力 |
| 轻型护盾模块 | 能量传导液、高容量电池 | 物流/战斗通用护盾模块 | 提高矿点和补给线存活 |
| 热能激光 | 能量传导液、高容量电池、透镜 | 能量武器 | 克制装甲但引入热量管理 |
| 重型履带底盘 | 高频振荡器、强化钢板、传动组件 | 重型战斗底盘 | 提供攻坚单位 |
| 中级逻辑板 | 高频振荡器、能量传导液、进阶电路 | 更多逻辑插槽 | 允许并列目标选择、防守、撤退规则 |
| 生命值判断 | 进阶电路、基础传感器 | 条件：自身生命值/护盾低于阈值 | 支持撤退、维修、保守进攻 |
| 前线补给点 | 强化钢板、高容量电池、建设质料 | 中距离物流中继建筑 | 支撑远离基地的资源点和战线 |
| 基础自卫武器模块 | 高频振荡器、轻型武器组件 | 物流机器人可装的小型自卫模块 | 让物流机器人不再完全依赖护航 |

阶段 Boss 门槛建议：

- 装甲 Boss 考验武器克制和重型底盘。
- 能量/空中 Boss 考验补给、防守和目标优先级。
- 资源点开始远离基地，玩家需要前线补给点和更强物流机器人。

## 阶段 3：战场回收

开启条件：击败干扰型巢穴 Boss，获得量子处理器。

阶段目标：让战场掉落物、敌方残骸和远距离物流成为主线的一部分。

| 科技节点 | 需求材料 | 解锁内容 | 设计作用 |
| --- | --- | --- | --- |
| 硅晶加工 | 量子处理器、精炼硅、能量传导液 | 集成处理器 | 支撑高级逻辑与精密武器 |
| 高级逻辑板 | 量子处理器、集成处理器、高容量电池 | 更多逻辑插槽、高级条件 | 允许多阶段战术脚本 |
| 广播与接收 | 量子处理器、集成处理器 | 动作/条件：广播信号、接收信号 | 支持小队协同和多蓝图配合 |
| 目标标记器 | 量子处理器、传感器阵列 | 条件/动作：锁定高价值目标 | 对抗治疗、干扰、远程敌人 |
| 装甲回收底盘 | 量子处理器、强化钢板、背包模块 | 可进入危险区域的物流底盘 | 回收我方与敌方机器人掉落物 |
| 残骸牵引模块 | 量子处理器、动力绞盘 | 物流模块：拖拽残骸 | 把战场回收做成明确能力 |
| 回收扫描器 | 量子处理器、传感器阵列 | 条件/行为：优先回收指定掉落物 | 支持自动筛选高价值战利品 |
| 巡航导弹 | 量子处理器、爆炸物、目标标记器 | 高级远程武器 | 奖励目标标记与补给链 |
| 远程投送塔 | 量子处理器、高容量电池、强化钢板 | 向指定前线投送机器人或物资 | 降低长距离地图带来的等待感 |

阶段 Boss 门槛建议：

- 敌人干扰普通传感器或打乱目标优先级。
- 玩家需要通过广播、目标标记和回收体系组织更稳定的战线。
- 地图上出现高价值残骸区，鼓励使用装甲物流机器人。

## 阶段 4：终局攻坚

开启条件：击败主脑级 Boss 或阶段性核心巢穴，获得主脑核心。

阶段目标：把此前积累的生产、蓝图、物流、规则和战术组合成终局攻坚方案。

| 科技节点 | 需求材料 | 解锁内容 | 设计作用 |
| --- | --- | --- | --- |
| 主脑解析 | 主脑核心、量子处理器、集成处理器 | 终局科技入口 | 明确进入最终阶段 |
| 终极 AI 矩阵 | 主脑核心、集成处理器、能量传导液 | 最高级逻辑板、更多规则容量 | 支持复杂多小队行为 |
| 堡垒级底盘 | 主脑核心、强化钢板、高容量电池 | 超重型战斗底盘 | 终局攻坚核心单位 |
| 远程投送无人机 | 主脑核心、量子处理器、背包模块 | 高级物流底盘/模块 | 支撑跨区域投送与回收 |
| 电弧链武器 | 主脑核心、能量传导液、高容量电池 | 群体能量武器 | 对抗大规模护卫 |
| 反物质奇点武器 | 主脑核心、稀有中间产物、目标标记器 | 终局高成本武器 | 作为高投入高回报的终局方案 |
| 高级诱饵模块 | 主脑核心、集成处理器 | 战术模块：吸引火力/误导目标 | 对抗 Boss 高伤害技能 |
| 最终攻坚协议 | 主脑核心、所有关键阶段研究 | 最终巢穴挑战入口 | 把科技树收束到胜利目标 |

## 数值升级

数值升级适合作为同阶段内的侧枝，不应承担主要门槛职责。它们让玩家有持续投入材料的空间，但不能替代新机制。

| 升级类型 | 示例 | 建议限制 |
| --- | --- | --- |
| 采集效率 | 采矿速度 +10%、矿机容量 +1 | 每阶段 1-2 级 |
| 生产效率 | 加工速度 +10%、装配耗能降低 | 不要压过新建筑价值 |
| 物流能力 | 物流速度、容量、投送冷却 | 与物流底盘科技配套 |
| 建设能力 | 建设质料产量、建筑修建速度 | 避免让前期等待过长 |
| 战斗属性 | 机器人生命、移动速度、武器伤害、冷却 | 小幅提升，用于平滑难度 |
| 规则能力 | 规则执行频率、感知范围、广播范围 | 尽量绑定逻辑板等级 |
| 视野能力 | 雷达范围、巢穴扫描精度 | 配合战争迷雾和地图推进 |

## 跳科技规则

推荐规则：

- 阶段之间不允许跳科技。
- 下一阶段入口必须消耗前一阶段 Boss 掉落物。
- 玩家可以提前探索高阶段区域，但不能绕过 Boss 门槛完成研究。
- 可以允许玩家冒险获得少量高阶段普通掉落物，但这些材料只能作为后续研究材料，不能直接解锁高阶段科技。
- 同一阶段内可以自由选择研究顺序，但关键节点之间保留清晰依赖。

这种做法的好处是：

- 地图推进和科技推进保持一致。
- Boss 不只是奖励来源，也是教学考试。
- 玩家仍有阶段内选择空间，不会完全变成线性清单。

## 逻辑插槽与规则数量

逻辑插槽不建议简单等同于“越多越强”。更好的关系是：

- 低级逻辑板：只允许少量规则，适合“集结后进攻”“血量低撤退”这类简单策略。
- 中级逻辑板：允许多个规则并存，支持进攻、防守、撤退、补给之间的切换。
- 高级逻辑板：支持广播、目标标记、队伍协同和回收优先级。
- 终级逻辑板：支持复杂多蓝图协同，但仍应有可理解的上限。

建议把规则数量上限设计成“认知容量”而不是纯数值压制：

- 每个逻辑插槽增加 1 条主动规则或 1 个规则块。
- 默认保底逻辑不占用插槽。
- 部分高级条件或动作可以占用更多逻辑容量。
- UI 中清楚显示当前蓝图的逻辑容量消耗。

## UI 表达

科技树 UI 建议采用阶段列结构：

- 每一列代表一个阶段。
- 阶段顶部显示 Boss 掉落物门槛。
- 已解锁、可研究、材料不足、Boss 门槛未满足四种状态要明显区分。
- 点击节点显示：材料需求、解锁内容、关联敌人、关联生产链。
- 数值升级与核心解锁分开显示，避免玩家误以为所有节点同等重要。

## 设计风险与修正建议

你的当前设想整体是成立的，但有几个需要提前控制的风险：

| 风险 | 可能问题 | 修正建议 |
| --- | --- | --- |
| Boss 掉落物过于硬 | 玩家卡 Boss 时完全停滞 | 每阶段保留若干非 Boss 支线升级，允许玩家变强后再挑战 |
| 科技节点过多 | 玩家不知道下一步该研究什么 | 每阶段只放少数核心科技，其他作为配方或数值升级挂在核心节点下 |
| 数值升级稀释机制 | 玩家只堆伤害和生命，不使用新系统 | Boss 机制必须要求新能力，例如集结、补给、回收、广播 |
| 物流科技太晚 | 远距离资源点让玩家感到疲惫 | 阶段 2 就给标准货运底盘和前线补给点 |
| 规则数量无限膨胀 | 蓝图 UI 和行为调试变得不可控 | 用逻辑板等级限制容量，用高级规则占用更多容量 |
| 阶段内完全自由 | 玩家可能先研究到无法使用的科技 | 核心节点之间保留少量依赖，例如热能激光依赖高容量电池 |

## 推荐结论

采用“Boss 锁阶段、阶段内分支科技树”的方案是合适的。它既能保护游戏节奏，又不会让玩家感觉每一步都被固定路线牵着走。

最关键的是每个阶段都应该回答一个新问题：

- 阶段 1：为什么默认攻击不够，为什么需要集结？
- 阶段 2：为什么需要补给线、装甲单位和武器克制？
- 阶段 3：为什么战场回收、广播和目标标记会改变战局？
- 阶段 4：如何把多个蓝图、物流链和复杂规则组合成最终攻坚体系？

只要科技树围绕这些问题展开，敌人、生产、物流和逻辑规则就会自然连成完整流程。
