# Logic Line 胜负、地图推进与失败惩罚设计

本文档用于回答三个核心问题：

1. 游戏是否需要战争迷雾？
2. 如果所有故事发生在一张大地图上，如何保证玩家按科技树阶段挑战敌巢与 Boss？
3. 当资源点远离玩家基地时，如何提供远距离物流与资源点防守能力？

推荐方案：

**单张大地图 + 分层威胁圈 + 轻战争迷雾 + 阶段巢穴门槛。**

也就是说，不采用完全切割的传统关卡地图，而是在同一张大地图上用地形距离、敌巢强度、资源类型、威胁等级和科技门槛形成自然阶段。

---

## 一、地图结构建议

### 1. 不建议早期使用完全分阶段切图

分阶段地图的优点是节奏可控，玩家不会过早碰到高阶敌人。但它会削弱自动化游戏最重要的感受：玩家亲手扩张基地，把远方资源点逐步接入自己的工业系统。

如果每个阶段都是独立地图，玩家会失去以下体验：

- 旧基地持续运转的成就感。
- 远距离物流网络逐渐变复杂的成长感。
- 前线推进后，旧战线变成后方的空间叙事。
- 敌巢被清理后，地图真的被玩家“驯服”的感觉。

因此更推荐使用一张大地图。

### 2. 推荐：同一张地图上的威胁圈

地图以玩家主基地为中心，向外分成多个环形或扇区式阶段。

| 区域 | 资源 | 敌人 | 目标 |
|---|---|---|---|
| 起始安全区 | 铁、铜 | 游荡小怪，低威胁巢穴 | 学会采集、生产、默认脑干 |
| 第一威胁圈 | 更多铁铜，小型特殊掉落 | 拾荒猎犬巢穴，小 Boss | 学会集结与第一条逻辑 |
| 第二威胁圈 | 晶体、煤、水 | 重甲犀牛、等离子飞蚊 | 学会伤害类型、热量、弹药补给 |
| 第三威胁圈 | 硅石、重金属 | 干扰塔与护卫群 | 学会特种机器人和协同逻辑 |
| 终局核心区 | 反物质、终极矿物 | 主脑巢穴、自适应巨兽 | 完成终局攻坚 |

玩家可以物理上提前走到更危险的区域，但会因为敌人强度、物流距离和资源需求而难以有效开发。

### 3. 阶段推进不是空气墙，而是软门槛

不要用“科技未解锁，所以不能进入该区域”的硬限制。更好的做法是让玩家可以冒险，但付出明显代价。

软门槛包括：

- 高阶巢穴守军击杀效率极低。
- 高阶敌人克制当前武器。
- 资源点距离过远，默认物流无法覆盖。
- 威胁值过高，前哨会频繁遭遇袭击。
- 缺少特定模块，导致掉落物无法有效回收。

这样玩家会自然意识到：我不是被系统拦住，而是我的工业和逻辑还没准备好。

## 二、战争迷雾设计

### 1. 不建议使用传统 RTS 全黑战争迷雾作为第一版

传统战争迷雾会带来很多额外系统负担：

- 探索单位。
- 视野刷新。
- 小地图信息层。
- 敌方行动隐藏规则。
- 玩家被偷袭时的公平性问题。

你的游戏核心是“自动化生产 + 机器人逻辑”，如果太早加入完整战争迷雾，玩家会把注意力转向侦察与地图操作，反而稀释核心体验。

### 2. 推荐使用轻战争迷雾

第一版建议采用三层地图信息：

1. **已扫描区域**：地图地形、资源点、敌巢位置可见。
2. **实时视野区域**：机器人、建筑、雷达覆盖范围内可见敌军动态。
3. **未知区域**：地图轮廓不可见，需要雷达塔、侦察机器人或推进到附近后揭示。

这种设计能保留探索感，但不会让玩家完全失去战略规划能力。

### 3. 敌巢可以提前显示“信号轮廓”

即使区域未完全探索，也可以让玩家看到远方有模糊敌巢信号：

- `弱敌巢信号`
- `重甲活动迹象`
- `高能反应`
- `干扰源`
- `主脑核心信号`

这样玩家知道阶段目标在哪里，但不知道具体守军配置。它比全黑地图更适合自动化游戏，因为玩家能提前规划生产方向。

## 三、胜利条件

### 1. 阶段胜利

每个阶段的胜利条件是：

1. 摧毁阶段性敌巢。
2. 击败该敌巢的小 Boss 或核心守军。
3. 回收关键掉落物。
4. 将掉落物运回基地或科技建筑。
5. 解锁下一阶段科技。

注意：**击败 Boss 不应立刻完成解锁，掉落物必须被物流系统带回。**

这样可以让物流机器人和战场回收系统成为核心闭环，而不是装饰系统。

### 2. 全局胜利

全局胜利条件是：

1. 摧毁终局主脑巢穴。
2. 击败自适应巨兽或主脑核心守卫。
3. 回收 `主脑核心`。
4. 完成终局科技或启动胜利工程。

可选终局目标：

- 激活轨道上传装置。
- 建造主脑解析器。
- 解锁无限沙盒模式。

## 四、失败条件与失败惩罚

### 1. 不建议轻易 Game Over

自动化游戏的失败惩罚不应频繁要求重开。更好的惩罚是让玩家的系统被打乱，但仍能修复。

推荐失败条件分为两层：

#### 软失败

玩家没有死亡，但扩张受挫。

例子：

- 远程采矿点被摧毁。
- 前线补给点被摧毁。
- 物流机器人损失，掉落物遗失在战场。
- 一波机器人生产链被打断。
- 威胁值上升，下一波袭击更强。

#### 硬失败

玩家真正输掉当前存档或当前战役。

例子：

- 主基地核心被摧毁。
- 主基地长期断电或关键系统瘫痪。
- 终局倒计时结束而玩家未能击败主脑。

第一版可以只实现软失败，硬失败作为后续战役模式目标。

### 2. 主基地保护规则

为了避免新手被滚雪球压死，主基地可以有阶段性保护：

- 第一次袭击不会摧毁主基地，只触发紧急护盾。
- 紧急护盾只能触发一次。
- 护盾触发后，系统提示玩家必须处理附近敌巢。

这和 `first-30-minutes.md` 的低惩罚失败相吻合。

## 五、敌巢推进规则

### 1. 敌巢不是单纯怪物生成器

敌巢应当具有三个状态：

1. **休眠**：玩家尚未接近，敌巢只维持本地守军。
2. **警觉**：玩家污染、噪音或单位接近后，敌巢开始积累威胁预算。
3. **进攻**：敌巢组建突袭小队，攻击玩家基地、资源点或前哨。

### 2. 敌巢目标选择

敌人袭击目标优先级建议为：

1. 最近的高威胁前哨。
2. 正在开采的远程资源点。
3. 前线补给点。
4. 机器人锻造厂。
5. 主基地。

这能让玩家感受到“扩张越远，越需要保护物流线和资源点”。

### 3. 击败敌巢后的地图收益

摧毁敌巢后，该区域进入“低威胁控制区”：

- 敌人刷新显著减少。
- 资源点可以稳定开采。
- 允许建造更高级的前哨或雷达塔。
- 该区域成为下一阶段扩张跳板。

这会让地图推进有实际空间收益。

## 六、远距离资源点与物流设计

### 1. 远距离资源点不应一开始就要求复杂物流

第一阶段资源点应在主基地附近，使用基地服务半径内的默认物流即可。

从第二阶段开始，资源点逐渐远离基地，玩家必须使用更高级的物流能力。

### 2. 远距离物流的分层解法

#### 前期：基地服务半径

- 主基地提供小范围自动物流。
- 微型搬运机器人不需要配置。
- 只服务铁、铜和基础工厂。

#### 中期：前线补给点与标准货运机器人

- 玩家在远程资源点附近建造 `前线补给点` 或 `小型仓库`。
- 标准货运底盘机器人负责在资源点、仓库、工厂之间搬运。
- 建筑施工也依赖物流机器人投料。

#### 后期：装甲回收车与远程投送

- 装甲回收车进入高危区域回收残骸和 Boss 掉落物。
- 远程投送塔/远程投送无人机负责把小型关键物资投送到前线。
- 重型资源仍需要普通物流路线，避免投送系统取代物流挑战。

### 3. 远距离物流的设计原则

- 小型高价值物品可以远程投送。
- 大宗基础资源必须走实体物流。
- 重型残骸必须由回收车拖拽。
- 战场掉落物必须被带回基地才算真正获得。

这样可以防止“远程投送”让整个物流系统失去意义。

## 七、资源点防守设计

### 1. 不需要传统防御塔作为核心防守手段

目前设计中没有防御塔，这是合理的。因为游戏核心是生产机器人和编写逻辑，如果防御塔过强，玩家会自然转向塔防玩法。

因此资源点防守应依赖：

- 巡逻机器人。
- 前哨集结点。
- 自动补兵的机器人锻造厂。
- 物流机器人修复与补给。
- 雷达预警与战斗报告。

### 2. 推荐的资源点防守结构

一个成熟资源前哨包含：

1. 采矿机。
2. 小型仓库。
3. 前线补给点。
4. 小型机器人锻造厂或部署口。
5. 集结点。
6. 雷达塔。

当敌人袭击时：

- 雷达塔提前发现敌人。
- 前哨锻造厂或主基地持续投送作战机器人。
- 作战机器人在集结点组成防守小队。
- 物流机器人补给弹药、回收残骸、修复建筑。

### 3. 防守逻辑示例

前哨防守机器人可以使用这类规则：

* `If 发现敌人 -> Then 前往前哨集结点`
* `If 周围友军数量 >= 3 -> Then 攻击最近敌人`
* `If 建筑 HP < 50% -> Then 优先保护受损建筑`
* `If HP < 30% -> Then 返回补给点`

物流机器人可以使用这类规则：

* `If 前线补给点弹药不足 -> Then 运送弹药箱`
* `If 建筑受损 -> Then 运送维修材料`
* `If 周围敌人数量 > 2 -> Then 撤离`

## 八、推荐地图阶段设计

### 第一阶段：起始盆地

**目标：** 建立基地、生产第一批机器人、摧毁第一个拾荒猎犬巢穴。

**资源：** 铁、铜。

**敌人：** 拾荒猎犬、小型巢穴守卫。

**地图特征：**

- 基地附近资源充足。
- 敌巢距离近，但不会一开始主动摧毁玩家建筑。
- 没有真正的远距离物流压力。

**阶段解锁：**

- 基础逻辑控制板。
- 集结点。
- 友军数量条件。

### 第二阶段：晶体荒原

**目标：** 建立第一个远程资源前哨，获取晶体、煤、水。

**资源：** 晶体、煤、水。

**敌人：** 重甲犀牛、等离子飞蚊。

**地图特征：**

- 资源点离主基地较远。
- 敌人开始袭击采矿点。
- 玩家需要前线补给点和标准货运机器人。

**阶段解锁：**

- 重型履带底盘。
- 热熔激光。
- 标准货运底盘。
- 前线补给点。

### 第三阶段：干扰高地

**目标：** 击败信号干扰塔，获得量子处理器。

**资源：** 硅石、重金属。

**敌人：** 干扰塔、重甲护卫、远程单位。

**地图特征：**

- 敌巢配置更像“考题”。
- 玩家需要特种侦察、锁定、导弹或绕后逻辑。
- 战场回收高价值掉落物变得重要。

**阶段解锁：**

- 高级战略逻辑板。
- 广播/接收信号。
- 装甲回收底盘。
- 远程打击协同。

### 第四阶段：主脑核心区

**目标：** 击败自适应巨兽或主脑核心守卫。

**资源：** 反物质、终极材料。

**敌人：** 自适应巨兽、终极巢穴、小型突袭群。

**地图特征：**

- 远距离物流压力极高。
- 敌人可能主动攻击玩家多处前哨。
- 玩家需要远程投送和多战线自动防守。

**阶段解锁或胜利：**

- 主脑核心。
- 终局科技。
- 沙盒模式或战役胜利。

## 九、设计结论

### 推荐答案

- **地图形式**：一张大地图，不切关。
- **阶段控制**：使用威胁圈、资源分布、敌巢强度和科技掉落作为软门槛。
- **战争迷雾**：采用轻战争迷雾，而不是传统 RTS 全黑迷雾。
- **资源点距离**：从第二阶段开始拉远，迫使玩家建设前哨和远程物流。
- **资源点防守**：不使用防御塔作为核心，依靠机器人、集结点、前线补给点和雷达预警。
- **失败惩罚**：以软失败为主，让玩家损失前哨、机器人、掉落物和时间，而不是频繁 Game Over。

### 核心理由

你的游戏核心不是“玩家手动守塔”，而是“玩家设计一套能自动扩张、自动防守、自动回收、自动反击的工业战争系统”。所以地图设计也应该服务这个目标：让玩家不断把自动化系统推向更远、更危险、更高价值的区域。
