这确实是一个非常棒的设计方向。引入“动能（Kinetic）”与“能量（Energy）”的双轴伤害体系，不仅能极大丰富战斗策略，还能完美反哺你的“自动化生产线”玩法——玩家为了生产不同维度的武器，必须在后方搭建截然不同的资源开采与加工流水线。

以下为你提供一份详细的《武器与底盘系统设计文档》，完全基于你的创意进行了深度扩充，并融入了科技树的推进节点。

### 一、 核心底层机制：双轴伤害维度

为了让两种伤害不仅是“数值换皮”，我们需要将它们与工厂的物流系统及战斗表现深度绑定：

1. **动能武器（物理/Kinetic）**
* **战斗特性**：具有明显的后坐力，命中时有概率造成“击退（Knockback）”或“硬直”效果。擅长击碎敌方的“能量护盾”。
* **生产/后勤限制**：**弹药消耗**。动能武器在战场上需要消耗实弹。这要求玩家在工厂端建立高效的“矿物提炼 -> 弹药制造 -> 战场补给”流水线。


2. **能量武器（魔法/Thermal & EM）**
* **战斗特性**：没有弹道飞行时间（如激光指哪打哪），擅长融化敌方的“重型物理装甲”，并能造成“系统过载（EMP）”等异常状态。
* **生产/后勤限制**：**热量与电力**。能量武器不需要补给弹药，但每次开火会积攒“热量（Heat）”或消耗机体电池。如果在UI逻辑面板中不写入“过热停火”的 If-Then 规则，机器人会因热量过高而宕机。

### 二、 底盘模块设计（Chassis Design）

底盘决定了机器人的基础血量、移动方式、**模块插槽数量** 以及**被动免疫能力**。

* **【前期】轻型轮式底盘 (Light Wheeled Chassis)**
* **插槽限制**：1个武器槽，1个基础逻辑槽。
* **速度与护甲**：极快，极低物理护甲。
* **特殊免疫**：无。
* **战略定位**：前期的廉价炮灰，造价极低，适合快速成型。


* **【中期】重型履带底盘 (Heavy Tracked Chassis)**
* **插槽限制**：3个武器槽，3个逻辑槽。
* **速度与护甲**：缓慢，极高物理护甲。
* **特殊免疫**：**免疫“击退”与“硬直”**。底盘极稳，受到重型动能武器攻击时不会被打断动作。
* **战略定位**：推进战线的主力肉盾，能够承载需要高稳定性的重机枪。


* **【后期】反重力悬浮底盘 (Anti-Gravity Hover Chassis)**
* **插槽限制**：2个武器槽，2个辅助装备槽，5个逻辑槽。
* **速度与护甲**：中等，自带基础能量护盾。
* **特殊免疫**：**免疫“地形减速”与“地雷陷阱”**。完全无视沼泽、酸液坑等复杂地形。
* **战略定位**：特种突击队，适合搭载高级能量武器从侧翼切入。


* **【大后期】堡垒级多足核心 (Fortress Hexapod Core)**
* **插槽限制**：5个武器槽，无限制逻辑槽。
* **速度与护甲**：极度缓慢，双抗拉满。
* **特殊免疫**：**免疫“EMP瘫痪”与“逻辑干扰”**。
* **战略定位**：移动的战区指挥中心或重型火力平台。

### 二点五、物流底盘与功能模块（Logistics Chassis & Utility Modules）

物流机器人也应纳入同一套“底盘 + 功能模块 + 逻辑板”的装配体系中。这样玩家未来不需要学习第二套单位规则：战斗机器人和物流机器人本质上都是机器人，只是底盘类型决定了可用插槽、战斗性能与任务定位。

#### 1. 设计原则

* **底盘决定职业边界**：物流底盘拥有背包栏位和搬运效率加成，但武器槽受限，无法搭载高性能武器。
* **前期默认免配置**：最开始的基础物流机器人不要求玩家配置模块或逻辑。它由主基地自动生产或提供，负责基地范围内的基础搬运，降低新手负担。
* **中后期逐步模块化**：当战场开始掉落资源、我方机器人残骸、敌方特殊材料后，物流机器人从“背景服务”升级为可设计的后勤单位。
* **后勤风险可被设计**：深入战场回收掉落物的物流机器人会暴露在敌方火力下，因此玩家可以通过护盾、自卫武器、远程投送等模块提高生存与效率。
* **不取代战斗机器人**：物流底盘的战斗参数应明显偏弱，不能通过堆武器变成廉价战斗单位。

#### 2. 物流底盘分类

* **【前期】微型搬运底盘 (Micro Hauler Chassis)**
* **插槽限制**：1个背包槽，0个武器槽，0~1个基础逻辑槽。
* **速度与护甲**：较快，极低护甲，低 HP。
* **特殊能力**：自动执行基地半径内的资源搬运任务。
* **战略定位**：前期默认物流单位。玩家不需要配置，用来理解“后勤机器人会自动服务产线”。
* **限制**：不能进入危险战场回收区，或进入后极易损毁。


* **【中期】标准货运底盘 (Standard Cargo Chassis)**
* **插槽限制**：2个背包槽，1个防御/工具槽，0~1个轻型自卫武器槽，1~2个逻辑槽。
* **速度与护甲**：中等速度，低护甲，中等 HP。
* **特殊能力**：可以执行战场边缘回收任务，例如搬运敌方掉落物、回收我方残骸、补给弹药箱。
* **战略定位**：将生产线和战场连接起来的核心后勤单位。
* **限制**：只能搭载轻型自卫武器，无法装备重机枪、激光、导弹等高性能战斗模块。


* **【后期】装甲回收底盘 (Armored Salvage Chassis)**
* **插槽限制**：3个背包槽，2个防御/工具槽，1个轻型自卫武器槽，2~3个逻辑槽。
* **速度与护甲**：较慢，较高护甲，中高 HP。
* **特殊能力**：拥有残骸牵引能力，可以回收较重的我方机器人残骸或 Boss 掉落物。
* **战略定位**：高危战场回收与前线补给。
* **限制**：战斗能力仍然很低，主要依赖护盾和路径逻辑生存。


* **【大后期】远程投送无人机底盘 (Remote Drop Drone Chassis)**
* **插槽限制**：1~2个背包槽，1个远程投送槽，1个防御槽，2~4个逻辑槽。
* **速度与护甲**：高速度，低护甲，可飞越部分地形。
* **特殊能力**：可以将小型物资或回收信标远程投送到指定区域，减少往返时间。
* **战略定位**：支持超远战线、导弹车补给、紧急回收信标部署。
* **限制**：容量低，不能搬运重型残骸，几乎没有正面战斗能力。

#### 3. 物流功能模块

* **背包模块 (Cargo Pack)**
* *功能*：增加携带容量。
* *设计用途*：让玩家用模块解决“单次运输太少”的效率问题。
* *可叠加性*：允许多个背包模块叠加，但每个背包模块会略微降低移动速度。


* **压缩货舱 (Compressed Cargo Bay)**
* *功能*：大幅增加容量，但提高制造成本与单位体积。
* *设计用途*：中后期用于搬运稀有掉落物、重型弹药箱或残骸碎片。


* **轻型护盾模块 (Light Shield Module)**
* *功能*：提供少量能量护盾，优先吸收远程伤害。
* *设计用途*：保护物流机器人穿越低强度战场火线。
* *限制*：护盾恢复需要回到基地或补给站。


* **装甲货壳 (Armored Cargo Shell)**
* *功能*：提高 HP 和物理抗性。
* *设计用途*：用于前线残骸回收。
* *代价*：降低速度和最大背包效率。


* **轻型自卫武器 (Light Self-Defense Weapon)**
* *功能*：低伤害、短射程，用于驱赶低级骚扰单位。
* *设计用途*：让物流机器人不至于被单个小怪完全白吃。
* *限制*：只能安装在物流底盘的自卫武器槽，伤害和射程远低于正规武器。


* **诱饵信标 (Decoy Beacon)**
* *功能*：短时间吸引敌方低级单位仇恨。
* *设计用途*：帮助物流机器人脱离危险区域，或者为战斗机器人争取时间。


* **远程投送模块 (Remote Drop Module)**
* *功能*：将携带的小型物资投送到指定信标、集结点或前线补给点。
* *设计用途*：减少长距离补给往返，支持后期导弹车、前线维修点和战场回收链。
* *限制*：投送有冷却时间，且不能投送大型残骸。


* **残骸牵引器 (Wreckage Tether)**
* *功能*：允许物流机器人拖拽我方报废机器人残骸或重型掉落物。
* *设计用途*：把“战场损失”转化为可回收资源，强化后勤价值。
* *限制*：拖拽期间速度大幅下降，容易成为敌人目标。


* **回收扫描器 (Salvage Scanner)**
* *功能*：扩大掉落物与残骸侦测范围。
* *设计用途*：帮助物流机器人自动选择更高价值的回收目标。

#### 4. 物流机器人与 If-Then 逻辑

前期物流机器人不要求玩家配置，但中期以后可以逐步开放简单逻辑。物流逻辑不应比战斗逻辑更复杂，它主要用于表达“何时冒险、何时撤退、优先捡什么”。

示例规则：

* `If 背包已满 -> Then 返回基地`
* `If HP < 40% -> Then 返回维修点`
* `If 发现稀有掉落物 -> Then 优先回收`
* `If 周围敌人数量 > 2 -> Then 释放诱饵并撤退`
* `If 前线补给点弹药不足 -> Then 远程投送弹药箱`

这类规则能把物流从纯粹后台搬运变成战场策略的一部分，但不会抢走战斗机器人的主角位置。

#### 5. 与战斗机器人底盘的差异

| 维度 | 战斗底盘 | 物流底盘 |
|---|---|---|
| 核心插槽 | 武器槽、逻辑槽、防御槽 | 背包槽、工具槽、防御槽 |
| 主要目标 | 击杀敌人、突破巢穴、保护战线 | 搬运、回收、补给、投送 |
| 武器限制 | 可搭载主武器与高级武器 | 只能搭载轻型自卫武器 |
| 逻辑复杂度 | 目标选择、集火、撤退、连招 | 路径选择、容量管理、危险规避 |
| 失败代价 | 损失战斗力与生产成本 | 掉落物丢失、补给中断、回收效率下降 |
| 战略价值 | 决定能否打赢考题 | 决定战线能否持续运转 |

#### 6. 设计风险与约束

* **风险：物流机器人过早变复杂。**
  * 约束：前期默认物流机器人不需要玩家配置，所有模块化物流只在战场回收开始后逐步开放。
* **风险：物流底盘被玩家改造成廉价战斗单位。**
  * 约束：物流底盘不能安装正规武器，轻型自卫武器的 DPS、射程和索敌能力都明显受限。
* **风险：战场回收变成繁琐清洁工作。**
  * 约束：默认回收逻辑必须足够好，玩家配置只用于高危、高价值、长距离场景。
* **风险：远程投送破坏物流挑战。**
  * 约束：远程投送只支持小型物资，且有冷却、能耗或投送信标限制。

### 三、 武器模块科技树与演进路线

武器根据触发方式分为**被动武器**（只要敌人在射程内就自动开火）和**主动武器**（必须由玩家编写 If-Then 逻辑来触发，通常伴随长冷却或高消耗）。

#### 1. 前期 (Early Game) - 基础材料阶段

* **被动：近战链锯 (Kinetic Chainsaw)**
* *伤害类型*：动能（极高）。
* *机制*：贴身物理撕裂。每次攻击附带“破甲”Debuff，削弱敌方物理抗性。适合装在轻型底盘上作为敢死队。


* **被动：轻型步枪 (Kinetic Rifle)**
* *伤害类型*：动能（中等）。
* *机制*：最基础的远程火力，射速慢，不耗电，弹药消耗极低。出厂自带，无需复杂逻辑。



#### 2. 中期 (Mid Game) - 引入电力与精炼材料

* **被动：重型转轮机枪 (Heavy Machine Gun)**
* *伤害类型*：动能（高）。
* *机制*：火力压制神器。但开火时会降低自身移动速度，且**弹药消耗极大**。考验玩家的自动化物流补给能力。


* **被动：热熔激光束 (Thermal Laser)**
* *伤害类型*：能量（中等，持续伤害）。
* *机制*：无视物理装甲。但会快速积攒机体热量。玩家需要开始在逻辑面板中学习编写：`If [自身热量 > 80%] -> Then [停止射击并后撤]`。


* **主动：冷冻凝胶喷射器 (Cryo-Gel Sprayer)**
* *伤害类型*：能量（极低）。
* *机制*：施加“冰冻”Debuff。大幅降低敌人的移动与攻击速度。如果对已过热的敌方机甲使用，会触发“热胀冷缩”引发额外真实伤害。



#### 3. 后期 (Late Game) - 引入芯片与高级逻辑配合

* **主动：EMP电磁脉冲发生器 (EMP Emitter)**
* *伤害类型*：能量（无直接伤害）。
* *机制*：造成“瘫痪”Debuff。击中敌方后，暂时切断敌方的逻辑运算器（使敌人的 If-Then 规则失效 3 秒），让敌方变成原地发呆的靶子。极度克制敌方的高级AI单位。


* **被动/辅助：战术目标锁定器 (Tactical Target Locker)**
* *伤害类型*：无。
* *机制*：扫描视野内血量最高或威胁最大的敌人，并为其打上“已锁定”标记。提升周围所有友军对该目标的命中率，并共享视野。


* **主动：动能巡航导弹 (Cruise Missile Launcher)**
* *伤害类型*：动能（毁灭性范围伤害）。
* *机制*：**射程无限，但只能对带有“已锁定”标记的敌人发射**。
* *逻辑高光*：强制要求玩家打出战术连招（Combo）。玩家需要生产两批机器人：一批装配悬浮底盘+锁定器冲锋陷阵；一批装配履带底盘+导弹在后方基地里待命。后方逻辑写为：`If [接收到'已锁定'信号] -> Then [发射导弹]`。



#### 4. 大后期 (End Game) - 终极自动化与清屏武器

* **被动：电弧闪电链 (Arc Lightning Emitter)**
* *伤害类型*：能量（高）。
* *机制*：攻击会在多个敌人之间弹射，是对付敌方海量虫群/低级兵海的最优解。但耗电量极其恐怖，必须搭配悬浮底盘或移动电池模块。


* **主动：反物质奇点炸弹 (Antimatter Singularity)**
* *伤害类型*：双轴混合真实伤害。
* *机制*：将目标区域的所有单位（不分敌我）吸入中心并造成巨额伤害。通常需要玩家牺牲一台寿命即将耗尽的“敢死队”机甲，冲入敌阵后触发：`If [寿命 < 5%] & [周围敌人 > 5] -> Then [引爆反物质炸弹]`。



### 总结：设计的巧妙之处

通过这套文档，你的游戏将形成非常深度的闭环：
前期玩家只需要造“采矿机->造步枪兵”无脑A上去；到了中期，遇到高护甲敌人，必须改造工厂去提炼新能源，造激光兵并学习“控制热量”的AI逻辑；到了后期，玩家不再是控制单一兵种，而是利用“锁定器 + 导弹”和“EMP + 链锯”打出基于 If-Then 规则的自动化兵种协同。这完美契合了“利用搭配组合战胜指定敌人”的核心愿景。

同时，物流机器人被纳入同一套底盘与模块体系后，后勤也能自然成长：前期它只是帮助玩家理解生产闭环的默认服务单位；中后期它会变成战场回收、弹药补给、远程投送和高价值掉落物运输的关键角色。这样可以让“生产线”和“战场”之间形成更强的双向联系，而不是战斗结束后资源凭空进入仓库。
