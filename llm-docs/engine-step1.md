这是一个极其清晰且务实的目标！作为有经验的程序员，从“灰盒 MVP”开始绝对是最聪明的做法，这能帮你迅速扫清 Godot 的底层架构盲区，而不会在美术和 UI 上浪费精力。

为了实现这个“单局斗蛐蛐”的闭环（配置逻辑 -> 部署 -> 交战 -> 回收），我为你制定了分为 5 个阶段的详细开发计划。每一步都建立在上一步成功运行的基础之上。

---

### 阶段零：基建与底层架构

在开始拼装机器人之前，先搭建好全局的后勤保障。

1. **创建项目结构：** 在文件系统中建立基础文件夹：`Scenes`（场景）、`Scripts`（脚本）、`Resources`（资源）。
2. **实现对象池 (Object Pool)：**
   * 创建一个脚本 `object_pool.gd`，实现我们之前讨论的 `get_instance` 和 `return_instance` 逻辑。
   * **关键步骤：** 进入 `项目 -> 项目设置 -> 自动加载 (AutoLoad)`，将 `object_pool.gd` 添加为全局单例（命名为 `ObjectPool`）。
3. **配置物理层 (Collision Layers)：**
   * 进入 `项目 -> 项目设置 -> 2D 物理`。
   * 将 Layer 1 命名为 `Team_A`，Layer 2 命名为 `Team_B`，Layer 3 命名为 `Projectiles`（子弹）。这在后续判断敌我时极其关键。

### 阶段一：制造“肉体”（基础场景与寿命）

目标：让一个方块能出现在屏幕上，并且活够时间后能自己“死”掉。

1. **创建机器人场景 (`robot.tscn`)：**
   * 根节点使用 `CharacterBody2D`。
   * 添加子节点：`ColorRect`（调成蓝色代表 A 队，大小设为 40x40，并将其居中）。
   * 添加子节点：`CollisionShape2D`（形状选 Rectangle，匹配方块大小）。
2. **实现状态与寿命：**
   * 为机器人挂载脚本 `robot.gd`，定义基础变量（如 `hp`, `speed`, `team`）。
   * 添加一个子节点 `Timer`，命名为 `LifespanTimer`。在 `_ready()` 中设置其等待时间并 `start()`。
   * 将 `LifespanTimer` 的 `timeout` 信号连接到 `robot.gd` 中，触发 `die()` 函数（调用 `ObjectPool.return_instance(self, ...)`）。
3. **阶段测试：** 在主场景中写个循环，从对象池里生成 10 个机器人，观察它们是否能在几秒后自动消失并成功回到池子。

### 阶段二：装配“雷达”（索敌系统）

目标：让机器人能感知到周围的环境和敌人。

1. **添加雷达节点：**
   * 在 `robot.tscn` 下添加子节点 `Area2D`，命名为 `Radar`。
   * 给 `Radar` 添加 `CollisionShape2D`（形状选 Circle，半径设为 150，这就是射程/视野）。
2. **配置碰撞掩码：**
   * 假设当前是 A 队，将其 `Radar` 的 `Collision Mask` 勾选为 B 队的层（Layer 2）。这样雷达只会对敌人产生反应。
3. **信号连接：**
   * 将 `Radar` 的 `body_entered` 和 `body_exited` 信号连接到脚本中，维护一个 `enemies_in_range` 的数组。
4. **阶段测试：** 摆放一个蓝方块和一个红方块，移动它们，看控制台是否能正确打印出“发现敌人”和“敌人离开”。

### 阶段三：植入“大脑”（AI 逻辑解析）

目标：让机器人能根据预设规则自己动起来。

1. **定义数据结构：**
   * 创建 `ai_rule.gd`（继承自 `Resource`），定义好 `Condition` 和 `Action` 的枚举（如我们之前讨论的）。
2. **创建 AI 控制器：**
   * 在 `robot.tscn` 下添加基础 `Node`，命名为 `AIController`，并挂载脚本。
   * 暴露导出变量 `@export var rules: Array[AIRule]`。
   * 添加 `TickTimer`（每 0.2 秒触发一次），并在 `timeout` 回调中遍历 `rules` 进行条件判断。
3. **实现基础动作：**
   * 在 Robot 脚本中实现几个供 AI 调用的基础接口：`move_towards(target)`，`stop()`。
4. **阶段测试：** 在编辑器里右键新建一个 `AIRule` 资源，设置为“始终(ALWAYS) -> 走向敌人(APPROACH)”。塞给蓝方块，运行游戏，蓝方块应该会像丧尸一样冲向红方块。

### 阶段四：武装“牙齿”（武器与伤害系统）

目标：实现真正的“互殴”。

1. **制作子弹场景 (`bullet.tscn`)：**
   * 根节点 `Area2D`，包含黄色的 `ColorRect` 和 `CollisionShape2D`。
   * 脚本中写简单的直线移动逻辑。
   * 连接 `body_entered` 信号：碰到敌人调用敌人的 `take_damage()`，然后自身放回对象池。
2. **制作武器挂载点：**
   * 在机器人下添加 `Marker2D` 作为枪口位置。
   * 在 Robot 脚本中添加 `fire()` 函数：从对象池获取子弹，设置初始位置为枪口，并赋予速度。
3. **完善伤害与死亡：**
   * 实现 `take_damage(amount)`，当 HP < 0 时，调用与寿命耗尽相同的 `die()` 逻辑。
4. **阶段测试：** 配置一条新规则：“发现敌人 -> 开火”。看两个方块互相射击，直到一方 HP 归零消失。

### 阶段五：组装斗兽场（MVP 闭环）

目标：跑通从配置到战斗结束的完整流程。

1. **创建主战场场景 (`main.tscn`)：**
2. **编写统筹脚本 (`game_manager.gd`)：**
   * 在代码中（不需要做 UI）硬编码创建两个配置好的数组，分别代表两队的机器人配置和 AI 规则。
   * 写一个 `start_match()` 函数，将两队机器人实例化到战场两边，并传入各自的规则。
3. **最终测试：** 按下 F6 运行 `main.tscn`，你应该能看到两拨方块被部署、自动索敌、互相开火、死掉的放回对象池、活着的寿命到了也放回对象池。MVP 达成！
