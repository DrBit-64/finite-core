# DDL 收口开发文档：最终 Boss、残骸拆解与锁定导弹闭环

本文档用于在项目 DDL 前把现有 post-MVP 内容收束成一个可展示的最终体验。

目标不是完整实现阶段 18-20 的所有系统，而是用最短路径做出一个清晰的高光闭环：

`残骸战场回收 -> 拆解残骸获得关键部件 -> 制造目标锁定机器人与巡航导弹机器人 -> 进入残骸战场后续区域 -> 一波又一波削弱并击败巨大移动 Boss`

## 一、收口目标

### 必须达成

- 游戏存在一个明确的阶段性最终 Boss。
- Boss 位于残骸战场之后的新区域，而不是塞回当前残骸战场内部。
- 当前残骸战场的三类产出物都进入最终机器人资源链：
  - `wreckage_scrap` / 残骸碎料。
  - `heavy_wreckage` / 重型残骸。
  - `salvage_data_core` / 残骸数据核心。
- 玩家可以通过残骸拆解获得新部件，并用这些部件制造最终战机器人。
- 玩家可以制造：
  - 目标锁定机器人。
  - 无限射程巡航导弹机器人。
- 导弹机器人不能自由攻击全图任意目标，只能攻击被锁定的目标。
- Boss 巨大、移动缓慢、血量高、自回血很慢，可以被多轮进攻逐步削弱。
- Boss 拥有多种攻击手段，并且每种攻击手段可以独立选择目标。

### 明确不做

以下内容在 DDL 收口版中不做完整系统化：

- 完整广播 / 接收网络。
- 任意蓝图之间的通用战术信号系统。
- 完整导弹弹药生产与补给链。
- Boss 复杂阶段变身。
- 完整 Boss 战复盘统计。
- 新区域全量美术打磨。
- 所有掉落物的完整存读档、合并、过期规则。

这些内容可以作为后续扩展，但不应阻塞最终展示闭环。

## 二、新区域设计

### 区域定位

新区域建议命名为：

- `interference_highland` / 干扰高地。
- 或 `wreckage_command_core` / 残骸指挥核心区。

推荐使用 `interference_highland`，因为它与现有 post-MVP 阶段 18 的方向一致，也能解释“为什么需要锁定与导弹”。

### 区域位置

区域应放在残骸战场之后。

建议空间关系：

- 起始盆地：中心区域。
- 晶体荒原：起始盆地右侧。
- 残骸战场：起始盆地左侧或左下侧。
- 干扰高地：残骸战场更深处，下一个连续区域。

如果当前地图中已经有预留的干扰高地区域，优先复用；否则在残骸战场外侧新增一片中等大小区域。

### 区域门槛

进入干扰高地不应只依赖普通资源。

推荐门槛：

- 玩家必须解锁 `stage3_battlefield_salvage`。
- 玩家必须带回至少 1 个 `salvage_data_core`。
- 玩家需要研究新的“远程锁定打击”科技。

这样残骸战场不只是刷资源区，而是通往最终 Boss 的前置准备区。

## 三、残骸拆解资源链

当前残骸战场已有三类核心产出：

- `wreckage_scrap`：普通残骸碎料。
- `heavy_wreckage`：重型残骸。
- `salvage_data_core`：残骸数据核心。

收口版需要把它们用于最终机器人，而不是只作为库存数字。

### 配方原则

残骸是战场高价值战利品，不是需要大量采集的基础矿石。

因此最终战相关拆解配方应采用 `1 -> n` 的数量关系：

- 1 份残骸产出多份中间部件。
- 辅助材料只作为加工催化或装配辅料，不应重新变成主要瓶颈。
- 玩家回收少量残骸后，应能立刻推进科技或制造至少一批最终战机器人。
- 不要设计成 `大量残骸 -> 少量部件`，否则 DDL 收口版会变成刷残骸。

### 新增中间资源

建议新增三个中间资源。

#### 1. `salvaged_alloy`

中文名：回收合金。

来源：

- 拆解 `wreckage_scrap`。

用途：

- 目标锁定机器人机体。
- 巡航导弹机器人机体。
- Boss 战前线消耗品的基础材料。

第一版配方：

`wreckage_scrap x1 -> salvaged_alloy x4`

#### 2. `recovered_servo_core`

中文名：回收伺服核心。

来源：

- 拆解 `heavy_wreckage`。

用途：

- 巨型重武器底盘。
- 巡航导弹发射支架。

第一版配方：

`heavy_wreckage x1 + reinforced_steel_plate x1 -> recovered_servo_core x3`

#### 3. `targeting_processor`

中文名：目标锁定处理器。

来源：

- 解析 `salvage_data_core`。

用途：

- 目标锁定机器人。
- 巡航导弹机器人。
- 远程锁定打击科技门槛。

第一版配方：

`salvage_data_core x1 + optical_lens x1 + high_capacity_battery x1 -> targeting_processor x3`

### 建筑归属

为了减少新增系统，拆解可以复用现有高级加工厂或研究终端。

推荐：

- `advanced_processor`：负责普通残骸拆解和重型残骸拆解。
- `research_terminal`：负责残骸数据核心解析。

如果实现时间紧，可以全部放进 `advanced_processor`，但 UI 文案要说明这是“残骸拆解/解析”。

## 四、最终科技：远程锁定打击

新增科技建议：

`stage3_remote_lock_strike`

中文名：阶段3：远程锁定打击。

解锁条件：

- 已研究 `stage3_battlefield_salvage`。
- 拥有或消耗 `targeting_processor x1`。
- 拥有一定数量的 `salvaged_alloy` 和 `recovered_servo_core`。

解锁内容：

- 目标锁定机器人。
- 巡航导弹机器人。
- 锁定目标模板。
- 锁定导弹模板。
- 干扰高地区域入口。

推荐科技文案：

> 从残骸数据核心中复原了远程制导协议。前线单位可以为高价值目标打上锁定信标，后方导弹平台随后进行超视距打击。

## 五、目标锁定机器人

### 单位定位

目标锁定机器人不是主力输出单位。

它的用途是：

- 进入危险区域。
- 尽可能给多个敌方目标添加 `locked` 标记。
- 持续补锁 Boss、高价值目标、远程敌人和护卫，使后方导弹机器人始终有合法目标。
- 没有可锁定目标时，才考虑撤离、绕行或远离危险区域。

### 建议 ID

`target_locker_robot`

### 推荐属性

- HP：110-160。
- 速度：110-130。
- 射程：260-340。
- 伤害：0 或极低。
- 寿命：120 秒。
- 逻辑容量：4。
- 标签：`combat`、`scout`、`locker`。

### 成本建议

- `salvaged_alloy x4`
- `targeting_processor x1`
- `optical_lens x2`
- `high_capacity_battery x1`
- `construction_mass x12`

### 最小行为实现

收口版不必实现完整广播。

推荐使用最小硬编码行为：

1. 机器人扫描范围内敌人。
2. 过滤已经处于有效 `locked` 状态、且剩余时间较长的目标。
3. 优先选择未锁定目标；若所有目标都已锁定，则选择 `locked` 即将过期的目标补锁。
4. 目标优先级建议为：`boss` / `high_value` / `jammer` > 远程输出敌人 > 召唤护卫 > 其它敌人。
5. 每次锁定需要 2 秒引导时间。
6. 引导期间目标必须仍然存活，且最好仍在锁定范围内。
7. 引导完成后给目标设置 `locked` 标记。
8. `locked` 持续 10 秒。
9. 锁定完成后立即寻找下一个可锁目标，而不是撤离。
10. 只有当扫描范围内没有可锁目标时，才进入撤离、绕行或远离危险区域逻辑。
11. 标记期间目标身上显示锁定视觉提示。

建议锁定状态字段：

```gdscript
locked_until_msec = Time.get_ticks_msec() + 10000
locked_by = locker_robot
```

建议锁定机器人自身状态字段：

```gdscript
locking_target = target
locking_finish_msec = Time.get_ticks_msec() + 2000
```

目标选择时可以把 `locked` 剩余时间低于 2 秒的目标视为“需要补锁”，避免导弹机器人刚准备开火时目标锁定过期。

### UI 表达

目标锁定机器人被点开时，应显示：

- 当前目标。
- 是否正在引导锁定。
- 引导剩余时间。
- 最近成功锁定数量。
- 扫描范围内可锁目标数量。

目标被锁定时，应显示：

- “已锁定”文字。
- 或一个小型锁定框 / 十字准星图标。
- 锁定剩余时间。

## 六、巡航导弹机器人

### 单位定位

巡航导弹机器人是最终战的后方输出单位。

它的核心规则：

> 射程无限，但只能攻击带 `locked` 标记的敌人。

### 建议 ID

`cruise_missile_robot`

### 推荐属性

- HP：90-130。
- 速度：45-60。
- 射程：99999。
- 伤害：160-260。
- 攻击间隔：8-12 秒。
- 溅射半径：80-120。
- 寿命：180 秒或无限。
- 逻辑容量：4。
- 标签：`combat`、`ranged`、`missile`、`long_range`。

### 成本建议

- `salvaged_alloy x8`
- `recovered_servo_core x2`
- `targeting_processor x1`
- `high_capacity_battery x2`
- `reinforced_steel_plate x6`
- `construction_mass x20`

### 最小行为实现

不做通用“收到信号”系统。

推荐使用全局目标查询：

1. 导弹机器人定期扫描所有敌方 `combat_target`。
2. 只筛选带 `locked` 标记且仍然存活的目标。
3. 优先攻击：
   - Boss。
   - 当前锁定剩余时间最长的目标。
   - 距离锁定机器人最近的目标。
4. 发射导弹。
5. 导弹飞行可以很快，但最好有明显视觉轨迹。

### 平衡原则

- 导弹不能单独解决问题，必须依赖锁定机器人。
- 导弹不能太快，否则 Boss 没有压迫感。
- Boss 自回血应慢于 1-2 台导弹机器人的持续输出，但能抵消少量零散攻击。

## 七、最终 Boss 设计

### Boss 概念

建议名称：

- `Wreckage Titan` / 残骸泰坦。
- 或 `Interference Behemoth` / 干扰巨兽。

推荐中文名：

`残骸泰坦`

它是一个巨大的移动缓慢机器人，像旧战场中被重新激活的战争机械。
它不主动进攻玩家基地，而是驻守在最终战场中心，作为玩家必须主动攻坚的阶段性目标。

### Boss 定位

- 不是敌巢建筑。
- 是一个巨大移动单位。
- 常驻最终战场中心，不主动向玩家基地或生产区推进。
- 可以在战场中心附近小范围巡弋、转向、调整站位或追击进入战场的目标。
- 血量很高。
- 自回血存在，但非常慢。
- 玩家可以多次进攻，逐步削血。
- 如果玩家攻击失败，Boss 不应快速回满。

Boss 的活动边界应由战场中心定义：

- 有一个 `home_position` 或 `arena_center`。
- 有一个 `arena_radius`，例如 500-800 px。
- Boss 不应离开该半径去追击玩家基地。
- 如果目标脱离战场范围，Boss 应停止追击并返回战场中心。
- Boss 战压力来自攻坚、控制和多武器反制，而不是基地被推平。

### 推荐属性

- HP：6000-12000。
- 速度：18-32。
- 护甲：`structure_armor` 或 `boss_armor`。
- 体型：普通机器人 4-6 倍。
- 自回血：每秒 3-8 点。
- 脱战不重置血量。
- 标签：`boss`、`high_value`、`jammer`、`frontline`。

### Boss 多武器系统

Boss 拥有四种攻击手段：

1. 机枪。
2. 激光。
3. EMP 炮弹。
4. 召唤近战护卫机器人。

重要要求：

> 每种攻击手段可以独立选择目标。

这意味着 Boss 不应只有一个 `current_target`。

推荐数据结构：

```gdscript
boss_weapon_slots = [
  {"id": "machine_gun", "target": null, "cooldown": 0.0},
  {"id": "laser", "target": null, "cooldown": 0.0},
  {"id": "emp_shell", "target": null, "cooldown": 0.0},
  {"id": "summon_guards", "target": null, "cooldown": 0.0}
]
```

每个武器槽独立执行：

1. 查找合法目标。
2. 根据自己的规则选择目标。
3. 冷却完成后攻击。

### 机枪

定位：

- 压制靠近 Boss 的普通机器人。
- 持续输出，但单发伤害不高。

目标选择：

- 优先最近的非肉盾单位。
- 如果有目标锁定机器人靠近，机枪可以优先打锁定机器人。

建议数值：

- 射程：220。
- 单发伤害：6-10。
- 攻击间隔：0.12-0.22 秒。

### 激光

定位：

- 对高血量或高护甲目标造成威胁。
- 迫使玩家不能只堆肉盾。

目标选择：

- 优先当前 HP 最高的玩家机器人。
- 或优先 `tank` 标签目标。

建议数值：

- 射程：320。
- 持续伤害：每秒 35-70。
- 持续时间：2-4 秒。
- 冷却：8-12 秒。

### EMP 炮弹

定位：

- 范围控制技能。
- 不是主要伤害来源，而是打断玩家进攻节奏。

目标选择：

- 优先机器人密集区域。
- 优先导弹机器人附近或集结点附近。

效果建议：

- 范围：120-180。
- 伤害：20-40。
- 附加状态：`emp_disabled`，持续 2-4 秒。
- 被 EMP 的机器人：
  - 暂停移动。
  - 暂停开火。
  - 或仅降低速度与攻速。

DDL 收口版如果不想改太多状态系统，可以先做简化：

- EMP 命中后调用 `movement_component.stop()`。
- 给机器人设置一个短暂 `emp_disabled_until_msec`。
- 在机器人攻击和移动前检查该时间。

### 召唤近战护卫

定位：

- 阻止玩家只用少量锁定机器人贴脸。
- 让导弹机器人需要后方支援。

目标选择：

- 优先召唤到锁定机器人附近。
- 或召唤到 Boss 与玩家主攻方向之间。

建议数值：

- 冷却：18-30 秒。
- 每次召唤：2-4 个小型近战护卫。
- 护卫类型可以复用 `scavenger_hound` 或新增弱化版 `boss_guard_drone`。
- 场上护卫数量应有上限，例如 10-12。

### Boss 自回血

Boss 自回血必须很慢。

设计目的：

- 防止玩家完全不进攻时 Boss 被零碎攻击磨死。
- 但允许玩家用多轮进攻逐步削弱。

推荐：

- 每秒回复最大生命值的 0.05%-0.10%。
- 以 8000 HP 为例，每秒 4-8 HP。
- 不做脱战快速回血。
- 不做阶段重置。

## 八、最终战流程

推荐玩家体验：

1. 玩家击败残骸战场指挥巢。
2. 玩家回收 `salvage_data_core`。
3. 玩家研究 `stage3_remote_lock_strike`。
4. 玩家拆解残骸获得：
   - `salvaged_alloy`
   - `recovered_servo_core`
   - `targeting_processor`
5. 玩家制造目标锁定机器人。
6. 玩家制造巡航导弹机器人。
7. 玩家进入干扰高地，看到驻守战场中心的巨大 Boss。
8. 普通机器人能削血，但效率很低且损耗大。
9. 目标锁定机器人靠近 Boss 并打上锁定标记。
10. 后方巡航导弹机器人自动发射远程导弹。
11. Boss 通过机枪、激光、EMP 和召唤护卫反制。
12. 玩家多轮生产、进攻、锁定、导弹打击，逐步击败 Boss。
13. Boss 死亡后触发最终胜利或 DDL 版本结局提示。

## 九、最低实现方案

如果时间极其紧张，按以下方式收口。

### 第 1 步：资源链

- 新增 3 个中间资源定义。
- 新增 3 条拆解配方。
- 新增最终科技 `stage3_remote_lock_strike`。
- 科技解锁目标锁定机器人和巡航导弹机器人。

### 第 2 步：目标锁定

- 给敌方单位/建筑添加可读写标记字典，或直接使用 metadata。
- 目标锁定机器人扫描并设置 `locked_until_msec`。
- 被锁定目标绘制锁定提示。

### 第 3 步：导弹机器人

- 新增导弹机器人单位配置。
- 新增导弹攻击逻辑：
  - 射程无限。
  - 只攻击 `locked` 目标。
  - 高伤害、长冷却。

### 第 4 步：Boss

- 新增巨大 Boss 机器人。
- 放置在残骸战场之后的新区域。
- 先实现：
  - 高 HP。
  - 慢速移动。
  - 慢回血。
  - 机枪。
  - 激光。
  - EMP 炮弹。
  - 召唤护卫。

如果多武器独立目标实现压力较大，也不要退回单一目标；至少保留每个武器独立选目标的函数接口。

### 第 5 步：胜利

- Boss 死亡后触发：
  - “干扰核心已摧毁”提示。
  - 暂停或显示胜利面板。
  - 记录最终事件。

## 十、可砍项

如果 DDL 前时间不足，可以砍：

- EMP 的复杂状态，只保留短暂停止移动/开火。
- Boss 激光的真实持续光束，只保留高亮线 + 周期伤害。
- 导弹弹药库存。
- 导弹飞行物理，只保留从导弹机器人到目标的快速弹体。
- 锁定信号 UI 动画，只保留目标上方文字和调试面板状态。
- 新区域完整美术，只要区域底色、边界、入口和 Boss 足够明确即可。

## 十一、验收标准

DDL 收口版只按以下标准验收：

- 玩家可以从残骸战场获得三类残骸产出物。
- 三类残骸产出物都能用于最终机器人相关资源链。
- 玩家可以研究远程锁定打击科技。
- 玩家可以生产目标锁定机器人。
- 玩家可以生产巡航导弹机器人。
- 目标锁定机器人能给 Boss 或高价值敌人添加锁定标记。
- 巡航导弹机器人不会攻击未锁定目标。
- 巡航导弹机器人能从远距离攻击已锁定目标。
- 新 Boss 位于残骸战场之后的新区域。
- Boss 是巨大移动机器人，不是静态敌巢，但它驻守战场中心，不主动进攻玩家基地。
- Boss 拥有机枪、激光、EMP 炮弹和召唤护卫四类能力。
- Boss 的不同攻击手段可以分别选择目标。
- Boss 血量很高，自回血很慢，玩家可以通过多轮进攻逐步削弱。
- Boss 死亡后触发明确的最终胜利或阶段通关提示。

## 十三、最终展示话术

可以把最终版本解释为：

> 前期玩家用生产自动化和基础蓝图击败敌巢；中期通过物流、热能武器和战场回收扩张；最终玩家把残骸战场回收来的材料拆解成锁定处理器与导弹部件，制造前线锁定机器人和后方巡航导弹平台，用自动化协同击败巨大残骸泰坦。

- 蓝图配置。
- 资源链推进。
- 地图阶段推进。
- 战术模板。
- 多机器人协同。
- Boss 作为系统考试。
