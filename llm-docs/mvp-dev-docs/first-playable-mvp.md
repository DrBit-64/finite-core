# First Playable MVP 开发范围冻结

本文档定义第一版可玩纵切版本的开发范围。目标不是做完整游戏，而是验证核心承诺：

`生产机器人 -> 默认 AI 失败 -> 修改逻辑规则 -> 集结后突破敌巢`

第一版必须让玩家在 15-30 分钟内完成一次完整体验：

1. 放置主基地。
2. 建立最小资源生产。
3. 建造机器人锻造厂。
4. 绑定一个机器人蓝图。
5. 生产机器人并自动补位。
6. 观察零散进攻失败。
7. 设置集结逻辑。
8. 集结后击败第一处敌巢。
9. 看到基础战斗报告。

## 一、MVP 设计原则

第一版只做“能验证核心玩法”的系统，不做完整模拟。

### 必须验证

- 生产线可以持续生成机器人。
- 机器人可以读取规则并行动。
- 默认脑干可以兜底战斗。
- 自定义规则可以改变战斗结果。
- 敌巢可以作为阶段目标。
- 战斗反馈可以告诉玩家为什么失败或成功。

### 暂不验证

- 完整物流机器人路径系统。
- 多蓝图产线队列。
- 大地图资源扩张。
- 完整科技树 UI。
- 完整建筑施工流程。
- 复杂敌人生态。
- 战场回收与远程投送。
- 任意时间范围复盘。

## 二、玩家流程

### 0. 开局

玩家进入一张小型测试地图。地图包含：

- 主基地初始位置。
- 1 个铁矿点。
- 1 个铜矿点。
- 1 个近距离敌巢。
- 1 个建议集结区域。

地图暂时不需要战争迷雾。敌巢可以直接显示，避免玩家把时间花在找目标上。

### 1. 建立基础生产

玩家放置或启用：

- 主基地。
- 铁矿采集点。
- 铜矿采集点。
- 基础加工厂。
- 机器人锻造厂。

第一版不要求真实物流机器人搬运资源。只要建筑处在主基地服务半径内，就可以访问主基地库存。

### 2. 绑定蓝图

机器人锻造厂绑定一个机器人蓝图。

第一版只需要一个默认战斗蓝图：

| 字段 | 内容 |
| --- | --- |
| 蓝图名称 | 基础步枪机器人 |
| 底盘 | 轻型战斗底盘 |
| 武器 | 轻型步枪 |
| 逻辑板 | 基础逻辑板 |
| 默认脑干 | 启用 |
| 自定义规则 | 可为空 |
| 生产耗时 | 12 秒 |
| 寿命 | 120 秒 |

蓝图可以读取当前已有 JSON 规则文件。若规则文件不存在、为空或解析失败，则只使用默认脑干。

### 3. 出生与部署

机器人从机器人锻造厂实体化出生。

锻造厂可以绑定一个地点作为集结点：

- 该地点是一个世界坐标。
- 可由玩家点击地图设置。
- 第一版也可以提供一个默认集结点。
- 机器人逻辑规则可以访问该地点。

机器人出生后不被强制脚本推向集结点，而是按照自身规则行动。

这条约束很重要：锻造厂只提供“出生”和“集结点引用”，真正的行为变化必须来自逻辑规则。

### 4. 自动补位

锻造厂持续尝试生产绑定蓝图。

推荐 MVP 规则：

- 锻造厂只绑定一个蓝图。
- 锻造厂有一个目标存活数量，例如 5 台。
- 当前存活数量低于目标时，自动排队生产。
- 资源不足时暂停，资源恢复后继续。
- 机器人死亡、寿命耗尽或失效后，计数减少，产线自动补位。

这样玩家不用手动补兵，可以专注观察规则效果。

### 5. 第一次失败

默认脑干行为：

`发现敌人 -> 接近敌人 -> 开火`

在没有自定义集结规则时，机器人会逐个前往敌巢并被守军消耗。

第一处敌巢的设计要求：

- 单台机器人无法突破。
- 2 台机器人很难突破。
- 3-5 台机器人同时进攻可以突破。
- 敌巢不会直接摧毁主基地，避免早期挫败。

### 6. 第一次规则修改

玩家新增一条集结规则。

推荐最小规则：

`If 周围友军数量 < 3 -> Then 前往锻造厂集结点`

或更稳定的版本：

`If 未到达锻造厂集结点 -> Then 前往锻造厂集结点`

然后依靠默认脑干在集结完成后继续寻敌和开火。

第一版不需要制作复杂“发起进攻”按钮。只要集结规则不再触发，默认脑干就会接管。

## 三、MVP 系统范围

## 1. 资源与抽象物流

第一版物流使用抽象搬运能力。

规则：

- 主基地拥有库存。
- 主基地有一个服务半径。
- 半径内建筑可以直接读写主基地库存。
- 采矿点按固定速率把资源加入库存。
- 加工厂按固定配方从库存扣资源并产出中间材料。
- 锻造厂从库存扣材料并生产机器人。

暂不做：

- 实体物流机器人。
- 搬运路径。
- 货物堆叠。
- 建筑输入输出口。
- 物流拥堵。

### 临时资源

第一版只需要 5 种资源：

| 资源 | 用途 |
| --- | --- |
| 铁矿 | 加工铁板 |
| 铜矿 | 加工铜线 |
| 铁板 | 机器人底盘和武器 |
| 铜线 | 逻辑板和基础电路 |
| 建设质料 | 建筑基础开销，由主基地被动生产 |

基础电路可以在第一版中先作为铜线的高级加工产物，也可以暂时省略，等蓝图配方需要时再加入。

## 2. 建筑

MVP 只需要 4 类建筑。

| 建筑 | 职责 | MVP 备注 |
| --- | --- | --- |
| 主基地 | 库存、服务半径、建设质料产出 | 必须实现 |
| 采矿点 | 产出铁矿/铜矿 | 可以先用固定建筑代替手动放置 |
| 基础加工厂 | 把矿物加工成铁板/铜线 | 可以自动循环 |
| 机器人锻造厂 | 绑定蓝图、生产机器人、设置集结点 | MVP 核心建筑 |

建筑生命值可以先只用于敌人袭击测试，不必实现完整修理系统。

## 3. 机器人

MVP 机器人必须具备：

- 队伍归属。
- HP。
- 移动速度。
- 寿命计时器。
- 索敌范围。
- 开火范围。
- 默认脑干。
- 自定义规则数组。
- 当前蓝图 ID 与蓝图版本。
- 出生锻造厂引用。
- 集结点坐标引用。

机器人死亡或失效原因至少包括：

- 被击杀。
- 寿命耗尽。
- 规则错误导致闲置超时。

过热和弹药耗尽可以先不进入第一版，文档与接口保留字段即可。

## 4. 规则系统

默认脑干永远存在。

执行顺序：

1. 按优先级从上到下检查自定义规则。
2. 第一条命中的规则执行动作，并结束本次 Tick。
3. 如果没有自定义规则命中，执行默认脑干。
4. 如果自定义规则解析失败，该规则跳过，并记录一次 `rule_failed`。
5. 如果规则目标不存在，该规则跳过，不阻塞默认脑干。

MVP 需要的条件：

| 条件 | 用途 |
| --- | --- |
| 距离目标小于 X | 既有战斗规则 |
| 自身 HP 低于 X% | 后续撤退测试，可选 |
| 周围友军数量小于 X | 第一阶段集结核心 |
| 未到达锻造厂集结点 | 更稳定的集结规则 |
| 发现敌人 | 默认脑干或显式战斗规则 |

MVP 需要的动作：

| 动作 | 用途 |
| --- | --- |
| 接近目标 | 默认脑干 |
| 开火 | 默认脑干 |
| 停止动作 | 调试与规则截断 |
| 前往锻造厂集结点 | 第一阶段核心动作 |

## 5. 敌巢与敌人

MVP 只需要一个敌巢类型。

敌巢行为：

- 固定位置。
- 拥有 3-5 个守军。
- 可以周期性补充守军，但补充速度很慢。
- 被摧毁后停止生成敌人。
- 击败后触发 MVP 胜利。

敌人行为：

- 发现玩家机器人后接近并攻击。
- 不需要复杂目标选择。
- 不需要主动扩张。
- 不需要真实生态预算。

第一版敌人只需要两类：

| 敌人 | 作用 |
| --- | --- |
| 拾荒猎犬 | 近战小怪，测试机器人是否会被围殴 |
| 小型巢穴核心 | 静态或慢速 Boss，测试集结后突破 |

## 四、临时经济数值

### 预期战斗结果

| 情况 | 预期结果 |
| --- | --- |
| 1 台机器人独自进攻 | 失败 |
| 2 台机器人零散进攻 | 大概率失败 |
| 3 台机器人同时进攻 | 勉强成功 |
| 5 台机器人同时进攻 | 稳定成功 |

这组结果是第一版平衡的核心。如果实测不符合，应优先调整敌巢护卫数量、机器人伤害和锻造厂生产速度。

## 五、复盘事件接口

第一版不需要完整复盘面板，但必须先埋事件接口。

推荐新增一个全局 `CombatEventLog` 或同等职责的节点/单例。

事件保存策略：

- 保留最近 10 分钟详细事件。
- 支持查询最近 1/5/10 分钟。
- 每条事件至少包含 `time`、`type`、`payload`。
- 第一版可以只在调试面板或控制台显示摘要。

### MVP 必须记录的事件

| 事件 | 触发时机 | 必要字段 |
| --- | --- | --- |
| robot_produced | 锻造厂完成生产 | robot_id、team、blueprint_id、blueprint_version、forge_id、cost |
| robot_lost | 机器人死亡或失效 | robot_id、team、blueprint_id、blueprint_version、reason、killer_type |
| enemy_killed | 敌人死亡 | enemy_id、enemy_type、killer_robot_id |
| rule_triggered | 规则命中并开始执行 | robot_id、blueprint_id、rule_id、rule_name、action |
| rule_failed | 规则解析或执行失败 | robot_id、blueprint_id、rule_id、reason |
| resource_spent | 生产或建造扣除资源 | resource_id、amount、reason |
| resource_gained | 采集或奖励获得资源 | resource_id、amount、source |
| nest_destroyed | 敌巢被摧毁 | nest_id、time_alive、reward |

### MVP 摘要面板

第一版只需要显示 5 分钟窗口：

| 指标 | 说明 |
| --- | --- |
| 生产机器人 | 统计 `robot_produced` |
| 损失机器人 | 统计 `robot_lost` |
| 击杀敌人 | 统计 `enemy_killed` |
| 规则触发次数 | 统计 `rule_triggered` |
| 从未触发规则 | 当前蓝图规则中计数为 0 的规则 |
| 资源消耗 | 统计 `resource_spent` |

显示形式可以很简陋，但必须能回答：

- 机器人是不是持续生产了？
- 为什么机器人没了？
- 集结规则是否真的触发？
- 击败敌巢前消耗了多少资源？

## 六、MVP 胜负条件

### 胜利条件

摧毁第一处敌巢。

胜利后显示：

- 用时。
- 生产机器人数量。
- 损失机器人数量。
- 集结规则触发次数。
- 敌巢奖励：初级感应线圈。

### 失败条件

第一版不做硬失败。

允许的软失败：

- 资源暂时不足，产线暂停。
- 机器人持续死亡，战斗报告提示“零散接敌”。
- 主基地被攻击后触发新手保护。

主基地可以有 HP，但降到 0 时先显示调试失败提示，不进入正式 Game Over 流程。

## 七、开发任务拆分

### Task 1：主基地与抽象库存

- 建立主基地库存。
- 实现服务半径。
- 实现建设质料被动产出。
- 支持资源增减事件。

### Task 2：采矿与加工

- 铁矿点、铜矿点按分钟产出。
- 基础加工厂自动加工铁板和铜线。
- 资源不足时等待。

### Task 3：机器人锻造厂

- 绑定一个蓝图。
- 绑定一个集结点坐标。
- 按目标存活数量自动补位。
- 完成生产后实例化机器人。
- 记录 `robot_produced` 与 `resource_spent`。

### Task 4：机器人默认脑干与规则兜底

- 默认脑干永远启用。
- 自定义规则优先于默认脑干。
- 规则失败不阻塞默认脑干。
- 支持前往锻造厂集结点。

### Task 5：第一敌巢

- 创建敌巢节点。
- 生成 4 个护卫。
- 护卫死亡后慢速补充。
- 敌巢被摧毁后触发胜利。

### Task 6：最小复盘

- 实现事件日志。
- 显示 5 分钟摘要。
- 显示规则触发次数和机器人损失原因。

### Task 7：首次体验调参

- 调整机器人生产耗时。
- 调整敌巢护卫数量。
- 调整友军数量阈值。
- 确保“零散失败，集结成功”稳定出现。

## 八、明确不做的内容

为了保护第一版开发范围，以下内容不进入 First Playable MVP：

- 实体物流机器人。
- 多种物流底盘。
- 远程投送。
- 战场回收。
- 弹药补给。
- 过热系统。
- 科技树完整 UI。
- 多敌巢地图推进。
- Boss 多阶段行为。
- 蓝图库管理。
- 多产线多蓝图队列。
- 存档读写。
- 建筑完整施工动画。
- 完整战争迷雾。

这些内容不是被废弃，而是等待第一条核心体验链成立后再加入。

## 九、通过标准

First Playable MVP 完成的标准：

1. 玩家可以从主基地开始生产基础资源。
2. 玩家可以建造或启用机器人锻造厂。
3. 锻造厂能绑定一个蓝图并持续生产机器人。
4. 机器人出生后能按规则行动。
5. 默认脑干在没有规则命中时能正常寻敌开火。
6. 没有集结规则时，机器人零散进攻敌巢会失败或高损耗。
7. 添加集结规则后，机器人能聚集并突破敌巢。
8. 摧毁敌巢后显示胜利摘要。
9. 复盘摘要能显示生产数、死亡数、规则触发数。

如果这 9 条成立，就说明游戏的核心想法已经被验证，可以进入下一轮扩展：实体物流、科技树、更多敌人、蓝图管理和完整复盘。
